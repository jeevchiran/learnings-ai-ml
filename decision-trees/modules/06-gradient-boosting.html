Bias vs. Variance — Two Different Problems

Module 5 showed that bagging and random forests reduce variance. When many high-variance trees disagree with each other, averaging their predictions cancels out individual errors and produces a stable model. This works because each tree is a low-bias, high-variance estimator: given enough depth, a single tree can fit the training data well, but its exact shape fluctuates wildly from dataset to dataset.

But what if the fundamental problem is bias? Consider a depth-1 decision tree — a stump. A stump makes one binary cut and assigns a constant prediction to each half. It is a drastically simplified model. No matter how many stumps you average, the resulting ensemble is still constrained by what a stump can express. If the true relationship requires more nuance than a single split, averaging stumps will not recover it. You have not reduced the systematic error; you have just averaged over it.

The bagging ceiling For an ensemble of M identical models, the bias of the average equals the bias of a single model. Bagging can reduce variance toward zero (as M grows), but it cannot reduce bias below the single-model floor. If every base learner is systematically wrong in the same direction, their average is just as systematically wrong.

To reduce bias, we need a different strategy: sequential learning. Rather than training many models independently and averaging, we train models one after another, where each new model specifically targets the errors made by all previous models combined. The ensemble grows additively — we sum the models rather than average them — and at each step we are actively correcting what was wrong before.

This is the key idea behind boosting.

The student analogy Imagine a student preparing for an exam. After the first attempt, she reviews every question she got wrong and studies those topics intensively. She retakes a similar exam, reviews her new mistakes, studies again, and repeats. Each iteration, the student improves specifically at her weakest areas. The final student is the accumulation of all those targeted study sessions. Boosting applies the same logic: each new model is a targeted correction of the current ensemble's weaknesses.
From variance reduction to bias reduction. Bagging builds many models in parallel, each trained on a bootstrap sample, and averages them to smooth out variance. Boosting builds models sequentially, each trained on the errors of the previous ensemble, and adds them together to reduce bias. The next section shows exactly how this works in the simplest case: regression with mean-squared error.

The Core Idea — Fitting Residuals

Start with regression to build concrete intuition. Suppose we have training data \{(x_i, y_i)\}_{i=1}^n and we want to build a predictive model F(x) by sequentially adding simple trees.

Step 0. Initialize with the simplest possible predictor: the mean of the targets.

F_0(x) = \bar{y} = \frac{1}{n}\sum_{i=1}^{n} y_i

The residuals are the errors this initial predictor makes:

r_i = y_i - F_0(x_i)

These residuals tell us exactly what the current model is getting wrong: positive residuals mean the model is under-predicting; negative residuals mean it is over-predicting.

Step 1. Train a tree h_1 to predict the residuals r_i (not the original targets!). Then update the ensemble:

F_1(x) = F_0(x) + \eta \cdot h_1(x)

where \eta \in (0,1] is the learning rate (also called shrinkage). Compute new residuals r_i = y_i - F_1(x_i) and repeat.

After M steps the final model is:

F_M(x) = F_0(x) + \eta \sum_{m=1}^{M} h_m(x)

Then h_2 corrects what h_1 left over, and so on. Each tree chips away at the remaining systematic error. Bias decreases with every iteration.

The General Framework — Functional Gradient Descent

The residual-fitting story above is a special case of a more general principle. For any differentiable loss function L(y, F(x)), gradient boosting minimizes:

\mathcal{L}(F) = \sum_{i=1}^{n} L(y_i, F(x_i))

over the space of functions F. Instead of gradient descent in parameter space (as in neural networks), we perform gradient descent in function space: each iteration adds a new tree that points in the direction of the negative gradient of \mathcal{L}.

The pseudo-residuals are the negative gradient of the loss with respect to the current predictions:

r_{im} = -\left[\frac{\partial L(y_i, F(x_i))}{\partial F(x_i)}\right]_{F=F_{m-1}}

We fit a regression tree h_m to these pseudo-residuals, then update F along the direction they indicate. This is gradient descent where the "parameter" being updated is the function value F(x_i) at each training point.

The Algorithm

Algorithm: Gradient Boosting

Input: Training data \{(x_i, y_i)\}_{i=1}^n, loss function L, number of trees M, learning rate \eta

  1. Initialize: F_0(x) = \arg\min_\gamma \sum_{i=1}^{n} L(y_i,\, \gamma) (e.g., \bar{y} for MSE; log-odds \log\frac{\bar{p}}{1-\bar{p}} for log-loss)
  2. For m = 1 to M:
    1. Compute pseudo-residuals: r_{im} = -\left[\partial L(y_i, F(x_i)) / \partial F(x_i)\right]_{F=F_{m-1}}
    2. Fit a regression tree h_m to the dataset \{(x_i,\, r_{im})\}_{i=1}^n, producing leaf regions R_{jm}
    3. Find optimal leaf weights: \gamma_{jm} = \arg\min_\gamma \sum_{x_i \in R_{jm}} L(y_i,\, F_{m-1}(x_i) + \gamma)
    4. Update: F_m(x) = F_{m-1}(x) + \eta \cdot h_m(x)
  3. Return F_M

Case 1: MSE Loss

For L(y, F) = \tfrac{1}{2}(y - F)^2, the pseudo-residuals are literally the ordinary residuals:

1
Write the gradient
r_{im} = -\frac{\partial}{\partial F}\left[\frac{(y_i - F)^2}{2}\right]_{F = F_{m-1}(x_i)}
2
Differentiate
\frac{\partial}{\partial F}\left[\frac{(y_i - F)^2}{2}\right] = \frac{1}{2} \cdot 2(y_i - F)(-1) = -(y_i - F)
3
Apply the negative sign
r_{im} = -\left[-(y_i - F_{m-1}(x_i))\right] = y_i - F_{m-1}(x_i) The pseudo-residuals for MSE are exactly the ordinary residuals y_i - \hat{y}_i. The residual-fitting story from Section 2 is not a heuristic — it is gradient descent on MSE loss.

Case 2: Binary Log-Loss

For binary classification, with \sigma(F) = 1/(1+e^{-F}) and loss:

L(y, F) = -\left[y\ln\sigma(F) + (1-y)\ln(1-\sigma(F))\right]
1
Rewrite using \sigma(F)
Note that 1 - \sigma(F) = \sigma(-F) and \frac{d}{dF}\sigma(F) = \sigma(F)(1-\sigma(F)).
2
Differentiate with respect to F
\frac{\partial L}{\partial F} = -\frac{y}{\sigma(F)} \cdot \sigma(F)(1-\sigma(F)) + \frac{1-y}{1-\sigma(F)} \cdot \sigma(F)(1-\sigma(F)) = -y(1-\sigma(F)) + (1-y)\sigma(F) = \sigma(F) - y
3
Negate to get pseudo-residuals
r_{im} = -\left[\sigma(F_{m-1}(x_i)) - y_i\right] = y_i - \sigma(F_{m-1}(x_i)) = y_i - \hat{p}_{m-1}(x_i) The pseudo-residuals are the difference between the true label and the predicted probability — exactly what you would intuitively want to correct.
Unified view In both cases — MSE and log-loss — the pseudo-residuals are of the form truth minus prediction. For regression it is y_i - \hat{y}_i; for classification it is y_i - \hat{p}_i. The gradient boosting algorithm is the same regardless of the loss. Only the definition of "residual" changes.

Shrinkage and Regularization

The learning rate \eta \in (0,1] scales each tree's contribution before adding it to the ensemble:

F_m(x) = F_{m-1}(x) + \eta \cdot h_m(x)

A smaller \eta means each tree contributes less. To achieve the same training error, we therefore need more trees (larger M). This is the shrinkage trade-off:

Learning rate \eta Trees needed M Regularization Generalization
Large (e.g., 1.0) Few Weak Often poor (overfitting)
Medium (0.1 – 0.3) Moderate Moderate Good balance
Small (0.01 – 0.05) Many Strong Often best, if compute allows
Empirical rule Lower \eta combined with higher M typically outperforms higher \eta with fewer trees, provided the computational budget allows it. In practice, \eta \in [0.05, 0.3] with early stopping on a validation set is the standard recipe. The interaction is explored visually in the second widget below.

Stochastic Gradient Boosting

Friedman (2002) introduced a simple but powerful regularization technique: at each iteration, draw a random subsample (fraction \text{subsample} \in (0.5, 1]) of the training data and fit h_m to that subsample only. This introduces randomness into the boosting process, analogous to the random feature selection in random forests.

Stochastic gradient boosting has two benefits. First, the reduced sample size speeds up each iteration. Second, the randomness acts as a regularizer, reducing the correlation between trees and often improving generalization, especially when \text{subsample} \approx 0.5.

Three regularization knobs in gradient boosting

  • Learning rate \eta — shrinks each tree's contribution. Primary regularization lever.
  • Tree depth — limits individual tree complexity. Shallow trees (depth 1–5) are common; they are weak learners that must cooperate.
  • Subsampling fraction — introduces randomness per iteration, reducing overfitting and correlation between trees.

All three are also hyperparameters that require tuning — covered in Module 7.

XGBoost — Second-Order Approximation

Vanilla gradient boosting uses only first-order information: the gradient (pseudo-residuals) tells us which direction to move, but not how far. XGBoost (Chen and Guestrin, 2016) incorporates second-order information — the Hessian — for a more accurate and regularized step.

At iteration m, XGBoost builds a Taylor expansion of the loss around the current predictions F_{m-1}(x_i):

L^{(m)} \approx \sum_{i=1}^{n} \left[ L(y_i,\, F_{m-1}(x_i)) + g_i \cdot f_m(x_i) + \frac{1}{2}\, h_i \cdot f_m(x_i)^2 \right] + \Omega(f_m)

where g_i = \partial_F L\big|_{F_{m-1}} is the gradient, h_i = \partial^2_F L\big|_{F_{m-1}} is the Hessian, and

\Omega(f) = \gamma |T| + \frac{1}{2}\lambda \|w\|^2

penalizes tree complexity: |T| is the number of leaves, w are the leaf weights, \gamma is the minimum gain needed to make a split, and \lambda is L2 regularization on leaf weights.

Dropping the constant term and grouping by leaf j (with I_j = set of sample indices in leaf j), the objective becomes a sum of independent quadratics in each leaf weight w_j. Setting the derivative to zero yields the closed-form optimal leaf weight:

w_j^* = -\frac{\sum_{i \in I_j} g_i}{\sum_{i \in I_j} h_i + \lambda}

Substituting back, the optimal gain for a candidate split (left set I_L, right set I_R, parent set I) is:

\text{Gain} = \frac{1}{2}\left[\frac{\left(\sum_{i\in I_L} g_i\right)^2}{\sum_{i\in I_L} h_i + \lambda} + \frac{\left(\sum_{i\in I_R} g_i\right)^2}{\sum_{i\in I_R} h_i + \lambda} - \frac{\left(\sum_{i\in I} g_i\right)^2}{\sum_{i\in I} h_i + \lambda}\right] - \gamma

A split is only made if Gain > 0 — the \gamma term acts as a minimum-gain threshold that prunes the tree during construction.

Concrete values for common losses

Gradients and Hessians by loss function

Loss Gradient g_i Hessian h_i Reduces to
MSE: \tfrac{1}{2}(y-F)^2 F(x_i) - y_i 1 Standard GB (h_i = 1 gives uniform curvature)
Log-loss \sigma(F) - y_i \sigma(F)(1-\sigma(F)) Adaptive step size; smaller steps near p = 0 or p = 1
Why does the Hessian help? The gradient tells us the slope of the loss at the current point. The Hessian tells us the curvature — how quickly the slope is changing. A steep, highly curved region needs a smaller step to avoid overshooting the minimum; a flat region can take a larger step. The optimal leaf weight w_j^* = -G_j / (H_j + \lambda) automatically scales the step by the curvature, achieving Newton-Raphson-style efficiency. This is why XGBoost often converges faster and to a better solution than standard gradient boosting.

Problem 1: Manual Boosting Step

Easy

We have 4 training points: x = [1, 2, 3, 4], y = [3, 5, 2, 8].

  1. Compute F_0 = \bar{y}.
  2. Compute the pseudo-residuals r_i = y_i - F_0 for all four points.
  3. A depth-1 stump splits at x \leq 2. The left leaf contains points with x \in \{1, 2\} and the right leaf contains x \in \{3, 4\}. What are the leaf predictions h_1(x) (mean residuals in each leaf)?
  4. With \eta = 0.5, compute F_1(x_i) for all four points.

(a) F_0 = (3+5+2+8)/4 = 18/4 = 4.5

(b) Pseudo-residuals: r = [3-4.5,\; 5-4.5,\; 2-4.5,\; 8-4.5] = [-1.5,\; 0.5,\; -2.5,\; 3.5]

(c) Left leaf (x \leq 2): indices {1,2}, mean residual = (-1.5 + 0.5)/2 = -0.5.
Right leaf (x > 2): indices {3,4}, mean residual = (-2.5 + 3.5)/2 = 0.5.

(d) F_1(x_i) = F_0 + 0.5 \cdot h_1(x_i):

  • F_1(1) = 4.5 + 0.5(-0.5) = 4.25
  • F_1(2) = 4.5 + 0.5(-0.5) = 4.25
  • F_1(3) = 4.5 + 0.5(0.5) = 4.75
  • F_1(4) = 4.5 + 0.5(0.5) = 4.75

New residuals: [-1.25,\; 0.75,\; -2.75,\; 3.25]. The split at x \leq 2 is still inadequate — x=3 and x=4 have very different targets (2 vs. 8) but end up in the same leaf. A better stump would split at x \leq 3 to separate them.

Problem 2: Binary Log-Loss Derivatives

Medium

For the binary log-loss L(y, F) = -y\log\sigma(F) - (1-y)\log(1-\sigma(F)) where \sigma(F) = 1/(1+e^{-F}):

  1. Derive the gradient: show that g_i = \partial L/\partial F = \sigma(F) - y.
  2. Derive the Hessian: show that h_i = \partial^2 L/\partial F^2 = \sigma(F)(1-\sigma(F)).
  3. For F = 0 (so p = 0.5) and y = 1, compute g_i and h_i.
  4. Using XGBoost's formula w^* = -G/(H+\lambda) with \lambda = 0: a leaf contains 3 positives (y=1) and 1 negative (y=0), all with F = 0. What is the optimal leaf weight?

(a) Gradient: Using d\sigma/dF = \sigma(1-\sigma):

\frac{\partial L}{\partial F} = -\frac{y}{\sigma}\cdot\sigma(1-\sigma) + \frac{1-y}{1-\sigma}\cdot\sigma(1-\sigma) = -y(1-\sigma) + (1-y)\sigma = \sigma - y

(b) Hessian: Differentiate \sigma(F) - y with respect to F:

\frac{\partial^2 L}{\partial F^2} = \frac{d\sigma}{dF} = \sigma(F)(1-\sigma(F))

(c) At F=0: \sigma(0) = 0.5.
g_i = 0.5 - 1 = -0.5.
h_i = 0.5 \times 0.5 = 0.25.

(d) 3 positives: each has g = -0.5, h = 0.25. 1 negative: g = 0.5 - 0 = 0.5, h = 0.25.

G = 3(-0.5) + 1(0.5) = -1.5 + 0.5 = -1.0 \qquad H = 4(0.25) = 1.0
w^* = -\frac{-1.0}{1.0 + 0} = 1.0

The optimal leaf weight is 1.0. Adding this to F=0 gives a new log-odds of 1.0, corresponding to \sigma(1.0) \approx 0.73 — a probability of 0.73 for the positive class, reflecting the 3:1 ratio of positives to negatives in this leaf.

Problem 3: Complexity Growth with Number of Trees

Hard

Gradient boosting with MSE loss, learning rate \eta = 1, and depth-1 stumps.

  1. Show that after M iterations, the model F_M can have at most M+1 distinct prediction values. Hint: each stump adds at most one new split threshold. How many regions can M thresholds create?
  2. Contrast this with a single depth-M decision tree, which can have up to 2^M leaves and therefore up to 2^M distinct predictions. What does this say about the representational efficiency of the two approaches?
  3. With \eta = 1 (no shrinkage), does gradient boosting still provide any regularization benefit over a deep single tree? Why or why not?

(a) Bound on distinct prediction values:

Each depth-1 stump h_m partitions \mathbb{R} into two regions using one threshold t_m. It takes one of two values: a_m for x \leq t_m and b_m for x > t_m.

After M trees, the model is F_M(x) = c_0 + \sum_{m=1}^M h_m(x). The function F_M changes value only at the thresholds t_1, t_2, \ldots, t_M. These M thresholds create at most M+1 contiguous intervals on the real line, each receiving a constant prediction. Therefore F_M takes at most M+1 distinct values.

Note: if some thresholds coincide, the actual count is fewer. So M+1 is an upper bound.

(b) Comparison with a depth-M tree:

A single depth-M binary tree can produce up to 2^M leaves, each with its own prediction. For M=10: the boosted ensemble of 10 stumps has at most 11 distinct predictions, while a depth-10 tree has up to 1024 leaves. The deep tree is exponentially more expressive.

This means gradient boosting with stumps requires many more trees (M \gg) to match the representational power of a single deep tree. However, the sequential construction means each stump is trained to correct the current error — a more targeted use of model capacity than the all-at-once greedy splitting of a deep tree.

(c) Regularization benefit with \eta = 1:

With \eta = 1, there is no shrinkage. However, using depth-1 stumps as base learners still provides regularization through model complexity: each tree is extremely weak, and the ensemble can only express piecewise-constant functions with at most M+1 pieces. A single deep tree at the same depth M would be far more complex (up to 2^M regions). So the regularization here comes from the shallow depth of the base learners, not from shrinkage. Setting \eta = 1 removes one regularization lever but the others remain.
