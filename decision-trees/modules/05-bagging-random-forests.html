Module 5: Bagging and Random Forests

The Problem Ensembles Solve

Module 4 identified a core tension in decision trees: a fully grown tree memorizes training data (high variance, low bias), while an aggressively pruned tree underfits (low variance, high bias). Pruning helps, but even a well-tuned single tree suffers from instability.

Instability means that small changes to the training data produce very different trees. Retrain on a dataset where one point is replaced by another drawn from the same distribution, and the greedy split at the root may change entirely — because a different feature suddenly offers the best Gini gain. Every subsequent split inherits this perturbation, and the final tree looks completely different. This is the consequence of greedy, top-down learning: the algorithm is sensitive to small changes in the data at the root, and that sensitivity cascades downward.

Contrast with k-nearest neighbors KNN is a stable learner. If you perturb one training point, only predictions for query points near that point change, and only slightly. A decision tree can have its entire structure reorganized by a single training point that happens to tip a split decision at the root. Instability is a structural property of greedy recursive partitioning, not a defect that better pruning can fully eliminate.

The Averaging Idea

The mathematical motivation for ensembles is simple. Suppose we have B independent, identically distributed random variables T_1, T_2, \ldots, T_B, each with variance \sigma^2. Their average has variance:

\text{Var}\!\left(\frac{1}{B}\sum_{b=1}^{B} T_b\right) = \frac{\sigma^2}{B} \xrightarrow{B \to \infty} 0

If we could train B independent trees and average their predictions, variance would vanish as B \to \infty. But there are two obstacles:

  • Not independent. All trees are trained on the same dataset, so they share structure and their predictions are correlated.
  • Not identically distributed. Different bootstrap samples produce trees with different distributions over the feature space.

Bagging and random forests address both obstacles — bagging introduces diversity through resampling, and random forests further decorrelate trees by randomizing feature selection at each split.

From instability to ensemble design. The insight is that variance can be reduced by averaging, but only if the things being averaged are not perfectly correlated. The entire engineering challenge of ensemble methods is reducing that correlation — making the base learners diverse enough that their errors cancel out when aggregated.

Bootstrap Sampling

A bootstrap sample of n observations is formed by drawing n times uniformly with replacement from the original n observations. Each draw is independent; any observation can be selected multiple times or not at all.

Expected Coverage: the 63.2% Rule

What fraction of the original training points appear in a typical bootstrap sample? For any single training point i, the probability it is not selected on a single draw is (1 - 1/n). Since there are n independent draws:

P(\text{point } i \text{ not selected}) = \left(1 - \frac{1}{n}\right)^n \xrightarrow{n \to \infty} e^{-1} \approx 0.368

Therefore, the probability that point i is included is approximately 1 - e^{-1} \approx 0.632. The expected number of unique training points in a bootstrap sample is:

E[\text{unique points}] = n \cdot P(\text{point included}) = n\left(1 - \left(1-\frac{1}{n}\right)^n\right) \approx n(1 - e^{-1}) \approx 0.632n

The remaining \approx 36.8\% of training points are called out-of-bag (OOB) observations for that bootstrap sample — they were never selected and therefore never influenced the tree trained on that sample.

Worked Example: n = 5 points {A, B, C, D, E}

Three independent bootstrap samples drawn from the same five points:

Sample Bootstrap Draw (size 5) Unique in-bag OOB
D1 A, A, C, D, A A, C, D (3 unique) B, E
D2 B, C, C, E, B B, C, E (3 unique) A, D
D3 A, B, D, D, E A, B, D, E (4 unique) C

Each sample produces a different effective training set, and each has a different OOB set. Trees trained on D1, D2, D3 will differ in structure because they see different subsets of the data.

Bagging — Bootstrap Aggregating

Leo Breiman introduced bagging (Bootstrap AGGregating) in 1996 as a general procedure for stabilizing any high-variance learner. Applied to decision trees, the algorithm is:

For b = 1 to B: a. Draw bootstrap sample D_b of size n from training data D (sample with replacement) b. Train decision tree T_b on D_b (grow to full depth — no pruning) Predict for new point x: Classification: majority vote of { T_1(x), T_2(x), ..., T_B(x) } Regression: mean of { T_1(x), T_2(x), ..., T_B(x) }

The ensemble prediction for regression is:

\hat{f}_{\text{bag}}(x) = \frac{1}{B} \sum_{b=1}^{B} T_b(x)

Trees are grown to full depth (maximum variance, minimum bias). The variance is controlled by averaging — not by pruning any individual tree.

Variance Reduction: the Derivation

Let each tree T_b have variance \sigma^2 and let any two trees T_i, T_j have correlation \rho (so \text{Cov}(T_i, T_j) = \rho\sigma^2). Then:

1
Expand the variance of the average
\text{Var}\!\left(\frac{1}{B}\sum_{b=1}^{B} T_b\right) = \frac{1}{B^2}\left(B\,\text{Var}(T_b) + B(B-1)\,\text{Cov}(T_i,T_j)\right)
There are B diagonal terms (each contributing Var(T_b) = σ²) and B(B−1) off-diagonal covariance terms.
2
Substitute Cov(Ti, Tj) = ρσ²
= \frac{1}{B^2}\left(B\sigma^2 + B(B-1)\rho\sigma^2\right) = \frac{\sigma^2}{B} + \frac{(B-1)}{B}\rho\sigma^2
3
Rearrange to the canonical form
\text{Var}\!\left(\frac{1}{B}\sum_b T_b\right) = \rho\sigma^2 + \frac{1-\rho}{B}\sigma^2
The first term is the variance floor: it does not shrink as B grows. The second term vanishes as B → ∞.
The correlation floor is the fundamental limit As B \to \infty, the ensemble variance approaches \rho\sigma^2. More trees cannot reduce variance below this floor. The floor is set by how correlated the trees are — which is set by the data and the learning procedure, not by B. This is why random forests introduce additional randomness to reduce \rho.

Out-of-Bag Error Estimation

Bagging provides a free cross-validation estimate. For each training point x_i, roughly 36.8% of the B trees were trained on bootstrap samples that did not include x_i. Let \mathcal{B}_i = \{b : x_i \notin D_b\} denote the set of trees for which x_i is OOB. The OOB prediction for x_i is:

\hat{f}_{\text{OOB}}(x_i) = \frac{1}{|\mathcal{B}_i|} \sum_{b \in \mathcal{B}_i} T_b(x_i)

The OOB error is then computed by comparing \hat{f}_{\text{OOB}}(x_i) to the true label y_i across all training points. This estimate is nearly unbiased for generalization error and eliminates the need for a held-out validation set — a significant practical advantage when data are scarce.

Random Forests — Decorrelating the Trees

Breiman (2001) refined bagging into random forests with one key modification: at each split, only a random subset of m_{\text{try}} features (out of the full p) is considered as candidates for the split. The best split is found among these m_{\text{try}} features only.

Why Feature Subsampling Reduces Correlation

In a bagged forest, all trees see the same overall feature distribution. If one feature is strongly predictive (say, feature X1 accounts for most of the class separation), nearly every tree will place X1 at or near the root. This makes the trees highly correlated — they all exploit X1 first, producing similar decision boundaries and prediction errors. The variance floor \rho\sigma^2 remains high.

By forcing each split to consider only m_{\text{try}} < p randomly chosen features, we ensure that sometimes the dominant feature is excluded from contention. Other features must be used in its place. Different trees end up relying on different features, discovering different aspects of the data. Their predictions become less correlated — \rho decreases — and the ensemble variance floor drops.

Choosing mtry

Task Breiman's default Practical advice
Classification m_{\text{try}} = \lfloor\sqrt{p}\rfloor Often works well out of the box; tune via cross-validation
Regression m_{\text{try}} = \lfloor p/3 \rfloor Try a range: p/10, p/5, p/3, p/2

With m_{\text{try}} = p, random forests reduce to bagging (all features considered at every split). With m_{\text{try}} = 1, each split is entirely random — maximum decorrelation but too much bias. The sweet spot is usually around \sqrt{p} for classification tasks.

The Full Random Forest Algorithm

Input: training data D = {(x_i, y_i)}, B trees, m_try features per split For b = 1 to B: 1. Draw bootstrap sample D_b (size n, with replacement from D) 2. Grow tree T_b from D_b: At each node with data subset S: a. Draw m_try features uniformly at random (without replacement) b. Find the best split among those m_try features only (using Gini impurity for classification, MSE for regression) c. Split S into left and right children d. Recurse on each child — no depth limit, no pruning 3. Store T_b Predict for new point x: Classification: y_hat = mode{ T_1(x), T_2(x), ..., T_B(x) } Regression: y_hat = (1/B) * sum{ T_b(x) } Optional: compute OOB error for free model validation

The Bias-Variance Tradeoff at the Ensemble Level

Restricting features at each split introduces a small amount of bias: we may occasionally miss the globally best split because the optimal feature was not in the randomly selected m_{\text{try}}. This is a real cost. But the variance reduction from decorrelation is typically much larger than the bias increase. The net effect on test error is almost universally positive — this is why random forests consistently outperform single decision trees across a wide range of datasets without careful hyperparameter tuning.

Random forests vs. bagging — when does decorrelation matter most? When one feature dominates (e.g., a feature with very high predictive power), bagging trees will be highly correlated and the variance floor will be high. Feature subsampling provides the largest benefit here. When features are roughly equally informative, the trees are already somewhat decorrelated by bootstrap sampling, and the gain from m_try is smaller.

Feature Importance

A useful byproduct of training a random forest is a ranking of feature importance. Two standard measures are used in practice.

Mean Decrease in Impurity (MDI)

For each feature j, sum the total impurity reduction (Gini gain) across every split on feature j across all trees, weighted by the fraction of training samples reaching that split:

\text{FI}(j) = \frac{1}{B} \sum_{b=1}^{B} \sum_{\substack{t \in T_b \\ \text{split on } j}} \frac{n_t}{n} \cdot \Delta G(t)

where n_t is the number of samples at node t and \Delta G(t) is the Gini gain at that split. Importances are normalized to sum to 1. A high \text{FI}(j) means feature j is used frequently and produces large purity gains.

MDI is biased toward high-cardinality features A feature with many distinct values (e.g., a continuous variable measured to many decimal places) has more candidate thresholds than a binary feature. More candidate thresholds means more chances to produce a winning split, inflating MDI even when the feature is not genuinely more informative. Permutation importance is less susceptible to this bias.

Permutation Importance

After training, for each feature j: randomly permute the values of feature j in the held-out (OOB or test) set, and measure the increase in prediction error. If shuffling feature j causes a large accuracy drop, the feature is important — the model depended on the real ordering of its values.

Formally, let \text{err}_{\text{OOB}} be the baseline OOB error, and \text{err}_{\text{OOB}}^{(j)} be the OOB error after permuting feature j. Then:

\text{PI}(j) = \text{err}_{\text{OOB}}^{(j)} - \text{err}_{\text{OOB}}

A large positive \text{PI}(j) means feature j matters. Permutation importance requires an extra pass through the OOB data for each feature, making it more expensive than MDI, but it is generally considered more reliable for feature selection.

MDI Advantages

  • Free — computed during training
  • No extra data pass needed
  • Consistent across runs

MDI Limitations

  • Biased toward high-cardinality features
  • Biased toward correlated features
  • Measures training-set importance

Practice

Problem 1: Out-of-Bag Probability Calculation

Easy

A random forest is trained with n = 1000 training samples and B = 500 trees. Approximately how many of the 500 trees will not have seen training sample #42 in their bootstrap sample? Show the calculation using the e^{-1} approximation.

The probability that a single bootstrap sample of size n = 1000 does not include point #42 is:

P(\text{point not selected}) = \left(1 - \frac{1}{1000}\right)^{1000} \approx e^{-1} \approx 0.368

Each of the B = 500 trees has this probability independently. The expected number of trees that did not see point #42 is:

E[\text{trees without point 42}] = 500 \times 0.368 \approx 184 \text{ trees}

These ~184 trees form the OOB committee for point #42. Their aggregate prediction is the OOB estimate for that point, computed without ever including it in training.

Problem 2: Variance Floor and Correlation

Medium

A random forest has trees with individual variance \sigma^2 = 0.16 and pairwise correlation \rho = 0.4.

(a) What is the ensemble variance at B = 10 trees?

(b) What is the variance floor as B \to \infty?

(c) To halve the variance floor, you need to halve \rho from 0.4 to 0.2. Suggest one mechanism from random forest design that would reduce \rho, and explain why it works.

(a) Ensemble variance at B = 10:

Using the formula \text{Var} = \rho\sigma^2 + \frac{1-\rho}{B}\sigma^2:

\text{Var}_{B=10} = (0.4)(0.16) + \frac{(1-0.4)}{10}(0.16) = 0.064 + \frac{0.6 \times 0.16}{10} = 0.064 + 0.0096 = 0.0736

(b) Variance floor:

\lim_{B \to \infty} \text{Var} = \rho\sigma^2 = (0.4)(0.16) = 0.064

Even with infinitely many trees, variance cannot drop below 0.064.

(c) Reducing ρ via feature subsampling:

Decreasing m_{\text{try}} (the number of features considered at each split) forces different trees to use different features as their primary splits. The dominant feature is excluded from some trees' consideration, so those trees must rely on other features — producing structurally different trees whose predictions correlate less. Reducing m_{\text{try}} from p (pure bagging) toward \sqrt{p} (standard random forest) is the primary lever for reducing \rho.

Halving the variance floor: (0.2)(0.16) = 0.032, which is indeed half of 0.064.

Problem 3: Full Variance Derivation and Limiting Cases

Hard

Starting from:

\text{Var}\!\left(\frac{1}{B}\sum_{b=1}^{B} T_b\right) = \frac{1}{B^2}\!\left[B\sigma^2 + B(B-1)\rho\sigma^2\right]

(a) Derive the formula \rho\sigma^2 + \frac{1-\rho}{B}\sigma^2 algebraically.

(b) Show that for \rho = 0 (independent trees), this equals \sigma^2/B.

(c) Show that for \rho = 1 (perfectly correlated trees), this equals \sigma^2 regardless of B, and interpret this result.

(a) Derivation:

\frac{1}{B^2}\!\left[B\sigma^2 + B(B-1)\rho\sigma^2\right] = \frac{\sigma^2}{B} + \frac{(B-1)\rho\sigma^2}{B}

Factor \sigma^2 and write \frac{B-1}{B} = 1 - \frac{1}{B}:

= \frac{\sigma^2}{B} + \left(1 - \frac{1}{B}\right)\rho\sigma^2 = \rho\sigma^2 + \frac{\sigma^2}{B} - \frac{\rho\sigma^2}{B} = \rho\sigma^2 + \frac{(1-\rho)\sigma^2}{B}

This is the canonical form. \checkmark

(b) Case ρ = 0 (independent trees):

(0)\sigma^2 + \frac{(1-0)}{B}\sigma^2 = \frac{\sigma^2}{B}

With zero correlation, averaging B trees reduces variance by exactly a factor of B — the classic result for independent random variables.

(c) Case ρ = 1 (perfectly correlated trees):

(1)\sigma^2 + \frac{(1-1)}{B}\sigma^2 = \sigma^2 + 0 = \sigma^2

The ensemble variance equals the individual tree variance regardless of B. This makes intuitive sense: if all trees are identical (perfectly correlated), averaging them produces the same prediction as a single tree — you are not gaining any new information. Growing more copies of the same tree achieves nothing. This is why decorrelation (the core contribution of random forests) is essential: high \rho makes the "ensemble" no better than a single model.
