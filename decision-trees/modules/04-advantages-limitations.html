Module 4: Advantages, Limitations, and Pruning

Why Does This Matter?

In Module 3 we built a working tree: we can choose splits, compute Gini gains, and grow a tree from root to leaf. The natural next question is whether we should grow the tree all the way. The answer is almost always no — and understanding why is the conceptual foundation for everything that follows, including ensembles.

Doctor analogy. A doctor who asks twenty questions before making a diagnosis may memorize every patient they have ever seen, giving perfect accuracy on past cases. But facing a new patient with a slightly different presentation, they fail — they are reciting memorized rules rather than reasoning. Conversely, a doctor who asks only two questions misses crucial nuance. The right number of questions is somewhere in between: enough to capture the real signal, not so many that noise gets memorized. Tree depth plays exactly the same role.

This module formalizes that intuition. We survey the genuine strengths of decision trees, then catalog their structural limitations, derive the bias-variance decomposition in the tree setting, and develop two principled strategies — pre-pruning and cost-complexity post-pruning — for finding a depth that generalizes.

Advantages

Advantages of Decision Trees

  • Interpretability. A tree can be visualized and explained to a non-expert. The path from the root to any leaf is a conjunction of human-readable rules: "Income > 50 000 AND Education = Graduate AND Age < 35 → Approve." Neural networks have no analogous representation; their decisions emerge from millions of interacting weights that resist any concise summary.
  • No feature scaling required. Trees split on thresholds, not distances. A feature with range [0, 1 000 000] receives identical treatment to one with range [0, 1] because the algorithm only asks "is the value above or below a threshold?" — it never computes a Euclidean distance or a dot product. Standardization and normalization are unnecessary preprocessing steps.
  • Handles mixed data types natively. Continuous features use threshold splits; categorical features use subset splits. Both can appear in the same tree. As shown in Module 3, the impurity calculation is identical in both cases — only the candidate split generation differs.
  • Captures feature interactions automatically. The condition "Income > 50 000 and Age < 30" is encoded by the path through two consecutive internal nodes. A linear model would require the analyst to manually add an Income × Age interaction term; the tree discovers it during training without any specification.
  • Robust to irrelevant features. If feature j carries no discriminative signal, it will never appear in any split, and its presence in the feature matrix causes no degradation. This stands in contrast to regularized linear models, which must explicitly shrink irrelevant coefficients toward zero.
  • Fast inference. Classifying a single sample requires traversing one root-to-leaf path: O(depth). For a balanced tree trained on n samples, the maximum depth is O(log n), giving O(log n) prediction time regardless of the number of features.

Limitations

Limitations of Decision Trees

  • High variance. Small perturbations in the training data can produce drastically different tree structures. Because the greedy algorithm commits to a split at each node without reconsidering earlier decisions, a different ordering of tied splits, or a single mislabeled point near a decision boundary, can flip the entire tree topology. Two training sets that differ by one point can yield trees with no split in common.
  • Overfitting. A fully grown tree can assign a distinct leaf to every training sample, driving training error to zero. A depth-d binary tree has at most 2^d leaves. Setting d = \lceil \log_2 n \rceil gives at least n leaves, enough for one sample each. The result is a lookup table, not a generalizing model.
  • Greedy algorithm with no global optimality guarantee. CART finds the locally optimal split at each node in polynomial time. Finding the globally optimal tree (minimum misclassification on the training set, over all possible trees of a given size) is NP-hard. The greedy choice can be suboptimal: a slightly impure first split might unlock a perfectly pure second-level partition, whereas the greedy "best" first split forecloses that possibility.
  • Axis-aligned decision boundaries only. Every split is of the form "feature j ≤ threshold," which is a hyperplane parallel to the j-th axis. If the true decision boundary is diagonal — for example, x_1 + x_2 > 5 — the tree requires exponentially many axis-aligned cuts to approximate it. The number of splits needed to achieve the same accuracy as a single oblique hyperplane grows as O(\epsilon^{-1}) for margin \epsilon.
  • Bias toward high-cardinality features. Features with many unique values have more candidate split points and therefore a higher chance of appearing optimal by chance. This bias is especially pronounced with information gain, where numerical attributes with many distinct values tend to score highest. Gini impurity partially mitigates this; the gain ratio correction in C4.5 addresses it more directly.

The Bias-Variance Tradeoff in Trees

For a model \hat{f} trained to approximate a true function f^*, the expected squared prediction error at a point x decomposes as:

\text{MSE}\!\left(\hat{f}(x)\right) \;=\; \underbrace{\!\left(\mathbb{E}\!\left[\hat{f}(x)\right] - f^*(x)\right)^{\!2}}_{\text{bias}^2} \;+\; \underbrace{\operatorname{Var}\!\left(\hat{f}(x)\right)}_{\text{variance}} \;+\; \sigma^2

The irreducible noise \sigma^2 cannot be reduced regardless of model complexity. The bias and variance terms trade off against each other as tree depth changes:

Shallow tree (depth near 0)

High bias: the model is too constrained to capture the true structure. A single-split stump approximates any function with at most two distinct predicted values. Variance is low because the model has few degrees of freedom and changes little across training sets.

Deep tree (depth near n)

Low bias on the training set: the model can memorize every label. Variance is high because tiny changes in the training data restructure the entire tree. The model is essentially interpolating noise.

Key insight A fully grown tree has zero training error but high variance. A stump has low variance but high bias. The right depth balances both — and that is precisely what pruning is designed to find.

In the tree setting, bias decreases monotonically with depth (more splits can always reproduce the training labels exactly), while variance increases monotonically (more splits means more sensitivity to individual training points). The optimal depth minimizes their sum at test time. Since neither the bias nor the variance is directly observable, we use held-out data or cross-validation as a proxy.

Pre-Pruning

Pre-pruning strategies stop the tree from growing before it reaches maximum depth. Each strategy imposes a constraint that prevents a split from occurring unless it meets a minimum benefit or minimum data threshold. The four standard strategies in scikit-learn are:

1
max_depth
Limits the length of any root-to-leaf path. Simple and widely used. The limitation is that it applies uniformly: a shallow branch that has already found pure leaves is stopped at the same depth as a branch that still has substantial impurity. It can terminate a beneficial split early in one part of the tree while allowing useless splits in another.
2
min_samples_split
Requires that a node contain at least m samples before it is eligible to be split. The default in scikit-learn is 2, which effectively disables this constraint. Setting m = 20 or 50 prevents the tree from creating splits based on small, possibly noisy subsets. It does not directly control the size of the resulting leaves.
3
min_samples_leaf
Requires that every leaf produced by a split contain at least m samples. Unlike min_samples_split, this controls the output size directly: a split is rejected if either of its children would have fewer than m samples. This is often more meaningful in practice because it prevents predictions from being based on single or very few observations.
4
min_impurity_decrease
Only performs a split if the weighted Gini (or entropy) gain is at least \delta. Formally, a split is accepted when:
\frac{N_t}{N}\,G(t) - \frac{N_L}{N}\,G(t_L) - \frac{N_R}{N}\,G(t_R) \;\geq\; \delta
where N_t, N_L, N_R are the sample counts at the node and its children and G denotes the impurity measure. This is the most semantically direct pre-pruning criterion: it directly says "the split must earn its keep."
Trade-off of pre-pruning Pre-pruning is computationally efficient — the tree never builds the nodes that would be pruned — but it can be myopic. A split that looks unpromising at depth k might unlock highly discriminative splits at depth k+1. Post-pruning, which grows the full tree first and then removes subtrees, sidesteps this problem at the cost of extra computation.

Cost-Complexity Post-Pruning

Grow the full tree T_0 first, then prune subtrees to minimize a regularized error objective. The cost-complexity criterion for a subtree T is:

R_\alpha(T) \;=\; R(T) + \alpha\,|T|

where R(T) is the training misclassification rate (or residual MSE for regression), |T| is the number of leaves in T, and \alpha \geq 0 is the complexity penalty per leaf. When \alpha = 0 the full tree is optimal; as \alpha grows, smaller trees become preferred.

The Effective Alpha and Pruning Order

For each internal node t, define the effective alpha at which pruning t (collapsing its entire subtree T_t to a single leaf) first becomes at least as good as keeping the subtree:

\alpha_t \;=\; \frac{R(t) - R(T_t)}{|T_t| - 1}

Here R(t) is the impurity of node t treated as a leaf (using its class distribution directly), and R(T_t) is the sum of leaf impurities in the subtree rooted at t. The denominator |T_t| - 1 is the number of leaves saved by collapsing.

The pruning algorithm processes nodes in order of increasing \alpha_t: the node with the smallest effective alpha is pruned first (it requires the least regularization to justify removal). This generates a nested sequence of trees:

T_0 \;\supset\; T_1 \;\supset\; \cdots \;\supset\; T_{\text{root}}

Each T_{k+1} is obtained from T_k by pruning the node with the current smallest effective alpha. Cross-validation is then used to select the best \alpha from this sequence.

Worked Example: Three-Level Tree

Consider the following tree with 3 internal nodes (t_0, t_1, t_2) and 4 leaves. The numbers below are training-set misclassification rates.

Practice

Problem 1 — Pre-pruning strategies for overfitting

Easy

A decision tree achieves 98% training accuracy but only 67% test accuracy. The gap indicates substantial overfitting. Name three pre-pruning strategies you would apply, explain what each parameter controls, and identify the likely structural cause of the gap.

Likely cause: The tree has grown too deep, creating leaves that memorize individual or very small groups of training points. The 31-point gap between training and test accuracy is a classic signature of high variance from excessive depth.

Three pre-pruning strategies:

  1. max_depth: caps the longest root-to-leaf path. Setting max_depth=5 or max_depth=10 (search via cross-validation) prevents the tree from encoding per-sample rules. This is the most direct lever for high-variance trees.
  2. min_samples_leaf: requires every leaf to hold at least m training samples. A leaf with m=20 represents a local average over at least 20 observations rather than a single point, directly reducing variance without imposing a hard depth limit.
  3. min_impurity_decrease: accepts a split only if the weighted impurity gain exceeds a threshold \delta. This prunes splits that contribute negligibly to purity but add complexity, targeting near-pure nodes that subdivide for statistical noise rather than true signal.

In practice, use cross-validation to tune all three jointly. Start with max_depth first as it is the most interpretable; refine with min_samples_leaf once a reasonable depth range is identified.

Problem 2 — Computing effective alpha

Medium

A tree has 3 internal nodes and 4 leaves. The root node t_0 has misclassification rate R(t_0) = 0.35 if converted to a leaf, and its subtree has R(T_{t_0}) = 0.12 with |T_{t_0}| = 4 leaves. Another internal node t_1 has R(t_1) = 0.28, R(T_{t_1}) = 0.18, |T_{t_1}| = 2. Compute \alpha_{t_0} and \alpha_{t_1}. Which node is pruned first as \alpha increases from 0?

Computing \alpha_{t_0}:

\alpha_{t_0} = \frac{R(t_0) - R(T_{t_0})}{|T_{t_0}| - 1} = \frac{0.35 - 0.12}{4 - 1} = \frac{0.23}{3} \approx 0.0767

Computing \alpha_{t_1}:

\alpha_{t_1} = \frac{R(t_1) - R(T_{t_1})}{|T_{t_1}| - 1} = \frac{0.28 - 0.18}{2 - 1} = \frac{0.10}{1} = 0.10

Since \alpha_{t_0} \approx 0.077 < \alpha_{t_1} = 0.10, node t_0 would be pruned first in the naive comparison. However, note that t_0 is the root and its subtree contains t_1. In practice, the algorithm prunes bottom-up: leaf-adjacent nodes first. After pruning t_1's sibling (if it also has a small effective alpha), t_0's subtree shrinks and its effective alpha is recomputed. The key result: the node with the smallest effective alpha among currently active internal nodes is pruned at each step. Here, if both nodes are leaf-adjacent, t_0 is pruned first (alpha 0.077 < 0.10).

Problem 3 — Proving alpha_t ≥ 0

Hard

Prove that for any internal node t in a decision tree, \alpha_t = (R(t) - R(T_t)) / (|T_t| - 1) \geq 0. Start by showing that R(T_t) \leq R(t) always holds (splitting can only reduce training error or keep it the same). Then explain why \alpha_t = 0 is possible and what it means about the split.

Step 1: Show R(T_t) \leq R(t).

Let node t contain n_t training samples with class distribution \{n_k\}. If treated as a leaf, the predicted class is the plurality class, so the training misclassification rate is R(t) = 1 - \max_k (n_k / n_t).

After any binary split of t into children t_L and t_R, each child is also predicted by its plurality class:

R(T_t) = \frac{n_L}{n_t}\,R(t_L) + \frac{n_R}{n_t}\,R(t_R) = \frac{n_L}{n_t}\!\left(1 - \max_k \frac{n_{L,k}}{n_L}\right) + \frac{n_R}{n_t}\!\left(1 - \max_k \frac{n_{R,k}}{n_R}\right)

We need to show this is at most R(t). By the definition of plurality prediction, any assignment of samples to two groups and predicting the majority within each group achieves at most as many errors as predicting the global majority. Formally, for any partition into L \cup R = \{1,\dots,n_t\}:

n_L \max_k n_{L,k} + n_R \max_k n_{R,k} \;\geq\; \max_k\!\left(n_{L,k} + n_{R,k}\right) = \max_k n_k

Dividing both sides by n_t and subtracting from 1 reverses the inequality: R(T_t) \leq R(t). Equality holds when the split does not separate any class more cleanly than the parent — for example, when both children have the same majority class in the same proportion as the parent.

Step 2: Conclude \alpha_t \geq 0.

Since R(T_t) \leq R(t), the numerator R(t) - R(T_t) \geq 0. The denominator |T_t| - 1 \geq 1 because t is an internal node with at least 2 leaves. Therefore \alpha_t = (R(t) - R(T_t)) / (|T_t| - 1) \geq 0.

Step 3: When is \alpha_t = 0?

\alpha_t = 0 iff R(t) = R(T_t), meaning the subtree achieves exactly the same training error as the leaf would. This happens when the split provides zero reduction in misclassification — both children predict the same class as the parent, or the split perfectly mirrors the parent distribution. In this case the subtree is useless from a training-error standpoint and should be pruned immediately (at \alpha = 0^+), because any positive complexity penalty, however small, makes the leaf strictly cheaper. This is called a degenerate split — it adds nodes without reducing error, representing pure overfitting to noise.
