Module 3: Gini Index Splits — Finding the Optimal Threshold

Building on Module 2: You now know how to measure node impurity using the Gini index and entropy. The next question is: given a continuous feature, how do we systematically find the split threshold that maximises impurity reduction?

1. The Core Problem of Splitting

For a continuous feature X with n training samples, there are up to n - 1 candidate split thresholds — one midway between each pair of adjacent sorted values. The algorithm must evaluate every candidate at every feature.

Naive Complexity

With p features and n samples, the naive approach evaluates O(n \cdot p) candidates per node, and each evaluation scans all n samples to recompute impurity. That is O(n^2 p) per node — unacceptably slow for large datasets.

Efficient Approach with Sorting

Sort the samples by feature value once: O(n \log n). Then sweep the sorted order, maintaining running class counts for left and right partitions. Each candidate threshold can be evaluated in O(1) by updating counts incrementally. Total cost per feature per node: O(n \log n) for the sort plus O(n) for the sweep.

Candidate Threshold Generation

Given sorted unique values v_1 < v_2 < \cdots < v_m, the m - 1 candidate thresholds are the midpoints:

s_i = \frac{v_i + v_{i+1}}{2}, \quad i = 1, \ldots, m-1

Using midpoints rather than raw values avoids ties and is invariant to the precise boundary location between classes.

Why midpoints? Any threshold in the open interval (v_i, v_{i+1}) produces the same partition. The midpoint is simply a convenient canonical representative. sklearn uses this convention.

2. The Gini Split Formula

Recall from Module 2 that the Gini impurity of a node t containing K classes with proportions p_1, \ldots, p_K is:

G(t) = 1 - \sum_{k=1}^{K} p_k^2

For a candidate split on feature j at threshold s, define:

  • t_L = \{x \mid x_j \le s\} — the left child node
  • t_R = \{x \mid x_j > s\} — the right child node
  • n_L, n_R — the sample counts in each child, with n_L + n_R = n

The Gini Gain (or reduction in weighted impurity) is:

\text{GiniGain}(j,\, s) \;=\; G(t) \;-\; \frac{n_L}{n}\,G(t_L) \;-\; \frac{n_R}{n}\,G(t_R)

The optimal split selects the feature and threshold that maximise this gain:

(j^*,\, s^*) = \operatorname*{arg\,max}_{j,\, s}\; \text{GiniGain}(j,\, s)

4. Continuous vs. Categorical Features

Continuous Features

Sort the n values: O(n \log n). Sweep once to evaluate all n-1 midpoints: O(n). Total per feature: O(n \log n). Always tractable.

Categorical Features

A feature with K categories has 2^{K-1} - 1 possible binary partitions into two groups. For K = 20 that is already 524,287 subsets — intractable by exhaustive search.

The Optimal-Ordering Shortcut for Binary Classification with Gini

For two-class problems, there is a remarkable result: the optimal binary partition of categories under the Gini criterion can always be found by sorting categories by their positive-class rate and then evaluating only contiguous prefix splits.

1
Setup

Let category c_k contain n_k samples, of which n_k^+ are positive. Define positive rate p_k = n_k^+ / n_k. The Gini impurity of category c_k as a pure group is p_k(1-p_k).

2
Weighted Gini of a left partition L

If left child collects some subset L \subseteq \{1,\ldots,K\}, then:

G(t_L) = 1 - \left(\frac{\sum_{k \in L} n_k^+}{n_L}\right)^2 - \left(\frac{\sum_{k \in L} n_k^-}{n_L}\right)^2
The weighted total impurity across both children is what we minimise over the choice of subset L.
3
Separability after sorting

Sort categories so p_1 \le p_2 \le \cdots \le p_K. It can be shown (Breiman et al., 1984 §4.6) that the objective

\min_{L \subsetneq [K],\, L \ne \varnothing}\;\frac{n_L}{n}\,G(t_L) + \frac{n_R}{n}\,G(t_R)

is achieved by a contiguous prefix L = \{1, 2, \ldots, m\} for some 1 \le m \le K-1. This reduces the search from 2^{K-1}-1 subsets to only K-1 ordered splits.

Intuition: mixing a low-rate category with a high-rate category always produces a noisier child than keeping them separate. The optimal partition never interleaves low and high rates.
4
Formal argument via convexity

The Gini weighted impurity of the left child can be written as a function of the left-child positive proportion \bar{p}_L = \sum_{k \in L} n_k p_k / n_L:

\frac{n_L}{n}\,G(t_L) = \frac{n_L}{n} \cdot 2\bar{p}_L(1-\bar{p}_L)

Because 2p(1-p) is concave in p, adding a category with p_k far from \bar{p}_L increases impurity more than adding one close to it. Keeping categories sorted and taking a prefix maintains the most extreme separation between child means — hence minimising total weighted impurity. The optimal L is always a contiguous prefix. \square

5
Complexity

Sort K categories: O(K \log K). Sweep K-1 prefixes: O(K). Total: O(K \log K) versus O(2^K) for brute force.

This shortcut does not hold for multiclass Gini or for entropy with more than two classes — exponential enumeration or heuristics are required there.

5. Gini vs. Entropy — When Do They Disagree?

Both Gini impurity and Shannon entropy are maximised at uniform class distributions and equal 0 at pure nodes. In practice they select the same split over 98% of the time (Breiman et al., 1984). Their functional forms over the binary positive probability p are:

G(p) = 2p(1-p), \qquad H(p) = -p\log_2 p - (1-p)\log_2(1-p)

Both are normalised to [0,1] by dividing by their maxima (1 and 1 respectively at p=0.5).

Key Practical Differences

Sensitivity near pure nodes

Entropy has a steeper gradient near p = 0 and p = 1: it assigns larger impurity reductions for near-pure splits. This can make it prefer a split that produces one very pure child slightly more than Gini does. The difference is numerically small but can occasionally change which feature is selected.

Computational cost

Gini requires only squaring and summing — cheap arithmetic. Entropy requires a \log call per class per candidate. On large datasets with many features, Gini is measurably faster and is the default in sklearn's CART implementation.

Concrete Example Where They Disagree

Consider a node with 20 samples (10P, 10F) and two candidate splits:

  • Split A: Left = {9P, 1F}, Right = {1P, 9F}.
  • Split B: Left = {10P, 4F}, Right = {0P, 6F}.

Problem 1 — Computing Gini Gain for a Single Split

Easy

A node has 10 samples: 5 positive, 5 negative. A proposed split produces:

  • Left child: 3 positive, 1 negative (4 samples total)
  • Right child: 2 positive, 4 negative (6 samples total)

Compute the Gini Gain for this split. Show all steps.

1
Parent Gini
G(\text{parent}) = 1 - \left(\frac{5}{10}\right)^2 - \left(\frac{5}{10}\right)^2 = 1 - 0.25 - 0.25 = 0.500
2
Left child Gini (3+, 1−)
G_L = 1 - \left(\frac{3}{4}\right)^2 - \left(\frac{1}{4}\right)^2 = 1 - 0.5625 - 0.0625 = 0.375
3
Right child Gini (2+, 4−)
G_R = 1 - \left(\frac{2}{6}\right)^2 - \left(\frac{4}{6}\right)^2 = 1 - \frac{1}{9} - \frac{4}{9} = \frac{4}{9} \approx 0.4\overline{4}
4
Weighted child impurity
\frac{4}{10}(0.375) + \frac{6}{10}(0.4\overline{4}) = 0.150 + 0.26\overline{6} = 0.41\overline{6}
5
Gini Gain
\text{GiniGain} = 0.500 - 0.41\overline{6} = 0.08\overline{3} \approx 0.083
This is a small but positive gain: the split does reduce impurity, though not by much.

Problem 3 — Proof: Optimal Categorical Split via Sorted Positive Rates

Hard

Prove that for binary classification, the optimal split point for a categorical feature under the Gini criterion is obtained by sorting categories in increasing order of their positive class rate p_k, then trying all K-1 contiguous prefix splits. Your proof should show that:

  1. The Gini objective can be written as a function of the left-child positive proportion.
  2. Any non-contiguous partition can be improved by swapping categories to make it contiguous.
  3. The optimal partition is therefore always a contiguous prefix of the sorted sequence.

Hint: Use the fact that 2p(1-p) is concave in p, and analyse what happens when you exchange a high-rate category in L with a low-rate one in R.

1
Express Gini in terms of child means

Let categories be indexed 1,\ldots,K with n_k samples and positive rate p_k. For a partition (L, R), the left child positive proportion is:

\bar{p}_L = \frac{\sum_{k \in L} n_k p_k}{\sum_{k \in L} n_k} = \frac{N_L^+}{n_L}

The weighted Gini of both children is:

\Phi(L) = \frac{n_L}{n} \cdot 2\bar{p}_L(1-\bar{p}_L) + \frac{n_R}{n} \cdot 2\bar{p}_R(1-\bar{p}_R)

We minimise \Phi(L) over all non-trivial subsets L.

2
Sort categories: WLOG assume p_1 \le p_2 \le \cdots \le p_K

Suppose for contradiction that the optimal L^* is non-contiguous: there exist indices i < j < k with i, k \in L^* but j \notin L^*. Consider swapping: remove k from L^* and add j to form L' (same size left child, just different membership).

3
Show the swap cannot increase \Phi

After the swap, the left positive count changes by n_j p_j - n_k p_k and the left total count changes by n_j - n_k. Define f(N^+, N) = 2(N^+/N)(1 - N^+/N). The objective is:

\Phi = \frac{N_L}{n}\,f\!\left(\frac{N_L^+}{N_L}\right) + \frac{N_R}{n}\,f\!\left(\frac{N_R^+}{N_R}\right)

Because p_j \le p_k (sorting assumption), the swap moves a lower positive-rate category into L and a higher one into R. This pulls \bar{p}_L down and \bar{p}_R up, making the two child means less extreme. By the concavity of 2p(1-p), the weighted sum of the two concave values is maximised when the means are equal and minimised when they are as far apart as possible.

\frac{n_L}{n}\,2\bar{p}_L(1-\bar{p}_L) + \frac{n_R}{n}\,2\bar{p}_R(1-\bar{p}_R)

Specifically, for fixed n_L, n_R: pushing \bar{p}_L toward 0 and \bar{p}_R toward 1 (or vice versa) reduces the weighted impurity because both concave terms decrease as their arguments move away from 0.5. Therefore keeping all low-rate categories in L is never worse than mixing.

4
Conclusion

By a sequence of such swaps (exchange argument), any non-contiguous optimal partition can be converted to a contiguous one without increasing \Phi. Therefore there always exists an optimal partition that is a contiguous prefix of the sorted-by-p_k ordering. We need only evaluate the K-1 prefix splits, reducing complexity from O(2^K) to O(K \log K). \blacksquare

This result holds for Gini only in the binary classification case. For entropy or multiclass settings, the argument breaks down because the objective is no longer a simple function of child means.
