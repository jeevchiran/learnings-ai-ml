Module 4: Mathematics of Logistic Regression

Bridge from Module 3: In Module 3 we used logistic regression as a classification tool -- fitting decision boundaries, selecting features via regularization, evaluating with ROC curves. Now we derive the mathematical foundations: why the sigmoid function? Why cross-entropy and not MSE? How does the optimization actually find the parameters?

Why Sigmoid? From Bernoulli to Logistic

We model y \mid \mathbf{x} \sim \text{Bernoulli}(p) where p = f(\mathbf{x}^T\boldsymbol{\beta}). We need f: \mathbb{R} \to (0,1), monotonic, and differentiable. The sigmoid is not an arbitrary choice -- it arises naturally from the exponential family.

1

The Bernoulli PMF in exponential family form:

p(y \mid \eta) = \exp\left(\eta y - \log(1 + e^\eta)\right)
where \eta = \log\frac{p}{1-p} is the natural parameter (log-odds).
2

The canonical link function connects \eta to the linear predictor:

\eta = \mathbf{x}^T\boldsymbol{\beta}
This is the defining choice of a GLM: set the natural parameter equal to the linear predictor.
3

Solve for p in terms of \eta:

p = \frac{e^\eta}{1 + e^\eta} = \frac{1}{1 + e^{-\eta}} = \sigma(\eta)
The sigmoid is the inverse of the logit -- it maps the natural parameter back to the mean parameter.
4

A useful derivative property:

\sigma'(z) = \sigma(z)(1 - \sigma(z))
This makes gradient computations elegant -- the derivative is expressed entirely in terms of the function itself.

Maximum Likelihood Estimation

Unlike linear regression, there is no closed-form solution. We use MLE: find \boldsymbol{\beta} that maximizes the probability of observing the training data.

1

Likelihood: For independent observations (x_i, y_i):

L(\boldsymbol{\beta}) = \prod_{i=1}^n p_i^{y_i}(1-p_i)^{1-y_i}
where p_i = \sigma(\mathbf{x}_i^T\boldsymbol{\beta}).
2

Log-likelihood:

\ell(\boldsymbol{\beta}) = \sum_{i=1}^n \left[ y_i \log p_i + (1-y_i)\log(1-p_i) \right]
The log transforms the product into a sum, which is easier to differentiate and numerically stable.
3

Substituting p_i = \sigma(\mathbf{x}_i^T\boldsymbol{\beta}):

\ell(\boldsymbol{\beta}) = \sum_{i=1}^n \left[ y_i \mathbf{x}_i^T\boldsymbol{\beta} - \log(1 + e^{\mathbf{x}_i^T\boldsymbol{\beta}}) \right]
Using \log\sigma(z) = z - \log(1+e^z) and \log(1-\sigma(z)) = -\log(1+e^z).

Cross-Entropy Loss: Why Not MSE?

The negative log-likelihood divided by n gives the cross-entropy loss:

J(\boldsymbol{\beta}) = -\frac{1}{n}\sum_{i=1}^n \left[ y_i \log(\sigma(\mathbf{x}_i^T\boldsymbol{\beta})) + (1-y_i)\log(1 - \sigma(\mathbf{x}_i^T\boldsymbol{\beta})) \right]

Why Not MSE?

If we used MSE with the sigmoid: J_{\text{MSE}} = \frac{1}{n}\sum(y_i - \sigma(\mathbf{x}_i^T\boldsymbol{\beta}))^2, the loss surface becomes non-convex due to the sigmoid's saturation regions. Gradient descent gets stuck in local minima.

Cross-entropy is convex in \boldsymbol{\beta} because -\log\sigma(z) is convex (second derivative is always positive). This guarantees a unique global optimum.

Computing the Gradient

1

Take the partial derivative of \ell with respect to \beta_j:

\frac{\partial \ell}{\partial \beta_j} = \sum_{i=1}^n \left(y_i - \sigma(\mathbf{x}_i^T\boldsymbol{\beta})\right) x_{ij}
Using \sigma'(z) = \sigma(z)(1-\sigma(z)) and the chain rule. The derivation simplifies beautifully.
2

In matrix form:

\nabla_{\boldsymbol{\beta}} \ell = \mathbf{X}^T(\mathbf{y} - \mathbf{p})
where \mathbf{p} = (\sigma(\mathbf{x}_1^T\boldsymbol{\beta}), \ldots, \sigma(\mathbf{x}_n^T\boldsymbol{\beta}))^T. This has the same form as the linear regression gradient -- the difference (\mathbf{y} - \hat{\mathbf{y}}) multiplied by \mathbf{X}^T. But here \mathbf{p} is a nonlinear function of \boldsymbol{\beta}, so there is no closed-form solution.

Optimization: Gradient Ascent for MLE

Since we maximize the log-likelihood (equivalently, minimize negative log-likelihood):

\boldsymbol{\beta}^{(t+1)} = \boldsymbol{\beta}^{(t)} + \alpha \cdot \mathbf{X}^T(\mathbf{y} - \mathbf{p}^{(t)})

Newton's Method and IRLS

Gradient descent uses only first-order information (the slope). Newton's method uses second-order information (the curvature) for faster convergence.

The Hessian

1

Second derivative of the log-likelihood:

\mathbf{H} = \frac{\partial^2 \ell}{\partial \boldsymbol{\beta} \partial \boldsymbol{\beta}^T} = -\mathbf{X}^T\mathbf{W}\mathbf{X}
where \mathbf{W} = \text{diag}(p_i(1-p_i)) is a diagonal weight matrix.
2

Newton's update rule:

\boldsymbol{\beta}^{(t+1)} = \boldsymbol{\beta}^{(t)} - \mathbf{H}^{-1}\nabla\ell = \boldsymbol{\beta}^{(t)} + (\mathbf{X}^T\mathbf{W}\mathbf{X})^{-1}\mathbf{X}^T(\mathbf{y} - \mathbf{p})
This is equivalent to solving a weighted least squares problem at each step -- hence "Iteratively Reweighted Least Squares" (IRLS).
3

Convergence: Newton's method has quadratic convergence (error squares each iteration) vs. linear convergence for gradient descent. Typically converges in 5-10 iterations vs. hundreds for GD.

Application: Credit Default Prediction

Working Through MLE by Hand

Consider 5 loan applicants with income (x_1) and debt ratio (x_2). Default labels: \mathbf{y} = (0, 0, 1, 1, 1)^T.

1

Initialize: \boldsymbol{\beta}^{(0)} = (0, 0, 0)^T. All probabilities start at \sigma(0) = 0.5.

Log-likelihood: \ell = 5 \cdot \log(0.5) = -3.47
2

Gradient at \boldsymbol{\beta}^{(0)}: \nabla\ell = \mathbf{X}^T(\mathbf{y} - 0.5\cdot\mathbf{1}). The gradient points toward the MLE.

3

After 3 iterations (learning rate \alpha = 0.5): \boldsymbol{\beta}^{(3)} \approx (-0.8, -1.2, 2.3)^T.

Interpretation: negative income coefficient means higher income reduces default probability. Positive debt-ratio coefficient means higher debt increases default risk. The model assigns the boundary where "the observed labels are most probable."

Regularized MLE: The Bayesian Interpretation

Adding a penalty to the log-likelihood has a natural Bayesian interpretation: regularization corresponds to placing a prior on \boldsymbol{\beta}.

1

MAP estimation: Instead of maximizing L(\boldsymbol{\beta}), maximize the posterior P(\boldsymbol{\beta} \mid \text{data}) \propto L(\boldsymbol{\beta}) \cdot P(\boldsymbol{\beta}).

2

L2 = Gaussian prior: If \beta_j \sim \mathcal{N}(0, \tau^2), then:

\log P(\boldsymbol{\beta}) = -\frac{1}{2\tau^2}\sum \beta_j^2 + \text{const}
This is exactly the L2 penalty with \lambda = \frac{1}{2\tau^2}. Smaller \tau = tighter prior = stronger regularization.
3

L1 = Laplace prior: If \beta_j \sim \text{Laplace}(0, b), then:

\log P(\boldsymbol{\beta}) = -\frac{1}{b}\sum |\beta_j| + \text{const}
The Laplace distribution has a sharp peak at zero, which is why L1 encourages exact sparsity. The probability mass concentrated at zero "pulls" coefficients there.

Multiclass Extension: Softmax

Generalize from binary (K=2) to K classes. Each class k has its own weight vector \boldsymbol{\beta}_k.

P(y = k \mid \mathbf{x}) = \frac{\exp(\mathbf{x}^T\boldsymbol{\beta}_k)}{\sum_{j=1}^K \exp(\mathbf{x}^T\boldsymbol{\beta}_j)}

Cross-Entropy for Multiclass

J = -\frac{1}{n}\sum_{i=1}^n \sum_{k=1}^K \mathbf{1}[y_i = k] \cdot \log P(y_i = k \mid \mathbf{x}_i)

Gradient for Softmax

\frac{\partial J}{\partial \boldsymbol{\beta}_k} = -\frac{1}{n}\sum_{i=1}^n \left(\mathbf{1}[y_i = k] - P(y_i = k \mid \mathbf{x}_i)\right)\mathbf{x}_i

The same "residual times features" structure persists. With K=2, this reduces to the binary logistic gradient.

Application: A/B Test Conversion Modeling

Logistic Regression Generalizes the Proportion Test

A standard A/B test compares conversion rates between two groups using a z-test on proportions. But logistic regression does the same thing -- and more:

\text{logit}(P(\text{convert})) = \beta_0 + \beta_1 \cdot \text{treatment}

Here \beta_1 is the log-odds ratio between treatment and control. Testing H_0: \beta_1 = 0 is equivalent to the proportion test. But the logistic framework lets you add covariates (user age, device type, country) to increase statistical power by reducing residual variance.

The Unifying Framework: Generalized Linear Models

Both linear and logistic regression are special cases of the GLM framework: an exponential family distribution for the response, a linear predictor \eta = \mathbf{x}^T\boldsymbol{\beta}, and a link function connecting E[y] to \eta. Poisson regression (for count data), gamma regression (for positive continuous data), and others fit the same structure. Neural networks extend this by stacking nonlinear transformations: each layer is a GLM whose output feeds the next.
