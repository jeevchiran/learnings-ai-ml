Module 5: Gradient Descent — Step by Step

Building on Module 2: The normal equation \hat{\boldsymbol{\beta}} = (\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y} is exact but O(p³). When p is large or \mathbf{X}^T\mathbf{X} is ill-conditioned, we iterate instead. This module walks through gradient descent step by step — the math, the code, and the intuition behind each move.

Why Not Always Use the Normal Equation?

Normal equation

Exact in one shot. Computing (\mathbf{X}^T\mathbf{X})^{-1} costs O(p³) — fine for p < 10 000, prohibitive beyond that.

Gradient descent

Iterative. O(np) per step. Scales to millions of features and examples. Foundation of all deep learning optimizers.

Six-Step Process

Given design matrix \mathbf{X} \in \mathbb{R}^{n \times (p+1)}, targets \mathbf{y} \in \mathbb{R}^n, and learning rate \alpha:

1
1
Initialize parameters

Start at \boldsymbol{\beta}^{(0)} — zeros or random small values. For convex losses like MSE, the starting point only affects convergence speed, not the final answer.

Python 
import numpy as np

n, p = X.shape                # n samples, p features (incl. bias col)
beta = np.zeros(p)          # β⁽⁰⁾ = [0, 0, ..., 0]
2
2
Compute predictions
\hat{\mathbf{y}} = \mathbf{X}\boldsymbol{\beta}
Python 
y_pred = X @ beta             # shape: (n,)
3
3
Compute loss
J(\boldsymbol{\beta}) = \frac{1}{2n}\|\hat{\mathbf{y}} - \mathbf{y}\|^2
The \tfrac{1}{2} is a convenience factor — it cancels the 2 produced by the chain rule when differentiating, yielding a clean gradient.
Python 
loss = np.mean((y_pred - y) ** 2) / 2
4
4
Compute gradient
\nabla J = \frac{1}{n}\mathbf{X}^T(\hat{\mathbf{y}} - \mathbf{y})
Why XT? Differentiating \frac{1}{2n}\|\mathbf{X}\boldsymbol{\beta}-\mathbf{y}\|^2 by the chain rule gives \frac{1}{n}\mathbf{X}^T(\mathbf{X}\boldsymbol{\beta}-\mathbf{y}). The \mathbf{X}^T maps the n-dimensional error vector back to p-dimensional parameter space. The \tfrac{1}{2} cancels the 2 from the power rule.
Python 
gradient = (1/n) * X.T @ (y_pred - y)  # shape: (p,) — same as beta
5
5
Update parameters
\boldsymbol{\beta}^{(t+1)} = \boldsymbol{\beta}^{(t)} - \alpha \cdot \nabla J
Step opposite the gradient (downhill). \alpha controls step size.
Python 
alpha = 0.01
beta  = beta - alpha * gradient
6
6
Repeat until convergence
Stop when loss change drops below a threshold ε, or after a fixed iteration budget.
Python — full training loop 
for iteration in range(1000):
    y_pred   = X @ beta
    loss     = np.mean((y_pred - y) ** 2) / 2
    gradient = (1/n) * X.T @ (y_pred - y)
    beta    -= alpha * gradient

    if iteration % 100 == 0:
        print(f"iter {iteration}: loss={loss:.4f}, β={beta}")
iter 0: loss=12.4832, β=[ 0.112 0.043] iter 100: loss= 2.1405, β=[ 2.301 5.984] iter 200: loss= 0.9812, β=[ 2.781 6.723] iter 300: loss= 0.8954, β=[ 2.931 6.902] iter 400: loss= 0.8878, β=[ 2.977 6.963]

Complete Example

Python — end-to-end 
import numpy as np

# Generate data: y = 3x + 7 + noise
np.random.seed(0)
X_raw = np.linspace(0, 10, 100)
y     = 3 * X_raw + 7 + np.random.randn(100) * 2

# Design matrix: [x, 1] for slope and intercept
X = np.column_stack([X_raw, np.ones(len(X_raw))])
n = len(y)

beta  = np.zeros(2)
alpha = 0.01

for _ in range(1000):
    y_pred   = X @ beta
    gradient = (1/n) * X.T @ (y_pred - y)
    beta    -= alpha * gradient

print(f"slope={beta[0]:.3f}, intercept={beta[1]:.3f}")
slope=2.998, intercept=7.027

Recovered true parameters (\beta_1 = 3,\; \beta_0 = 7) to 3 decimal places.

Learning Rate Effects

αBehaviourSymptom
\alpha \ll 1 — too small Tiny steps Loss decreases but needs thousands of extra iterations
Just right Smooth descent Loss drops rapidly then plateaus near minimum
\alpha > \frac{2}{\lambda_{\max}} — too large Divergence Loss oscillates or grows to ∞
Convergence guarantee: For convex J (true for MSE), GD with \alpha < \frac{2}{\lambda_{\max}} converges to the global minimum, where \lambda_{\max} is the largest eigenvalue of \frac{1}{n}\mathbf{X}^T\mathbf{X}.

Variants

Batch GD

Gradient over all n samples each step. Exact, stable, expensive for large n.

Stable Slow/epoch
Stochastic GD

Gradient from 1 sample per step. Noisy but fast; O(p) per update.

Fast Noisy
Mini-Batch GD

Gradient from k samples (32–512). Vectorised efficiency + reduced variance. Industry standard.

Balanced GPU-friendly
Python — mini-batch GD 
batch_size = 32
for epoch in range(100):
    indices = np.random.permutation(n)
    for start in range(0, n, batch_size):
        idx      = indices[start : start + batch_size]
        y_pred   = X[idx] @ beta
        gradient = X[idx].T @ (y_pred - y[idx]) / len(idx)
        beta    -= alpha * gradient

Practice

Problem 1: Manual update

Easy

For J(\beta) = (\beta - 5)^2, starting at \beta^{(0)} = 0 with \alpha = 0.1: compute \nabla J, \beta^{(1)}, and J(\beta^{(1)}).

∇J at β=0: 2(0-5) = -10

β⁽¹⁾: 0 - 0.1 \times (-10) = 1.0

J(β⁽¹⁾): (1-5)^2 = 16 vs J(0)=25 — improved by 9.

Problem 2: Gradient shape check

Medium

You have n=100, p=3 (including bias). State the shape of \mathbf{X}, \boldsymbol{\beta}, (\hat{\mathbf{y}}-\mathbf{y}), and \nabla J. Verify the matmul dimensions are compatible.

  • \mathbf{X}: (100, 3)
  • \boldsymbol{\beta}: (3,)
  • \hat{\mathbf{y}}-\mathbf{y}: (100,)
  • \mathbf{X}^T \cdot (\hat{\mathbf{y}}-\mathbf{y}): (3,100)·(100,) = (3,) = shape of β ✓

The gradient always has the same shape as β — one partial derivative per parameter.

Problem 3: Safe learning rate

Hard

For 1D data \mathbf{X} = [1, 2, 3]^T (no bias), compute the numerical safe learning rate bound \alpha < \frac{2}{\lambda_{\max}} where \lambda_{\max} = \mathbf{X}^T\mathbf{X}/n.

\mathbf{X}^T\mathbf{X} = 1 + 4 + 9 = 14, n=3, so \lambda_{\max} = 14/3 \approx 4.667.

Safe bound: \alpha < 2/4.667 \approx \mathbf{0.429}.

Quiz

Module 5 — Check Your Understanding

1. Why write the cost as \frac{1}{2n}\|\hat{y}-y\|^2 instead of \frac{1}{n}\|\hat{y}-y\|^2?

AIt makes the loss smaller and easier to interpret
BThe ½ cancels the 2 from the chain rule, giving a cleaner gradient \frac{1}{n}\mathbf{X}^T(\hat{y}-y)
CIt minimizes a different, better objective
DTo keep the learning rate from needing adjustment
Correct. Both forms share the same minimizer β. The ½ is purely cosmetic — it cancels the 2 from ∂/∂β of the squared term.
Both forms reach the same β. The ½ cancels the 2 produced by the power-rule derivative, keeping the gradient formula clean.

2. The gradient ∇J has shape:

AScalar
B(n,) — one value per sample
C(p,) — same shape as β
D(n, p) — full Jacobian
Correct. X^T maps (n,) errors to (p,) parameter space. One gradient entry per parameter.
X^T is (p,n). Multiplying by an (n,) error vector gives (p,) — same shape as β, so β -= α·∇J is valid.

3. Setting α beyond the safe bound causes:

AConvergence to a different minimum
BFaster convergence
CDivergence — each step overshoots further than the last
DOscillation that eventually settles
Correct. Beyond α = 2/λ_max, the step overshoots the minimum; the next step overshoots further — exponential divergence.
The bound α < 2/λ_max is exact for quadratic losses. Exceeding it causes each step to land further from the minimum than the previous one.

4. Mini-batch GD with batch_size = n equals:

ASGD
BBatch GD
CNewton's method
DThe normal equation
Correct. batch_size = n uses all samples per update — identical to full batch GD.
batch_size = n means every update uses the full dataset — that is batch GD. batch_size = 1 gives SGD.
