Schema Inspection

Before doing any analysis, you need to know what you're working with. Schema inspection tells you the column names, their data types, and whether they allow nulls. This is the first thing you should do with any new DataFrame.

  • df.printSchema() — prints the schema as a tree with column types and nullable flags
  • df.dtypes — returns column names and types as a Python list of tuples
  • df.columns — returns just the column names as a list

Descriptive Statistics

Once you know the schema, the next step is understanding the distribution of values. PySpark provides two methods for quick statistical summaries:

  • df.describe() — computes count, mean, standard deviation, min, and max for numeric columns
  • df.summary() — includes everything in describe() plus percentiles (25%, 50%, 75%)

Note that describe() does not show the median (50th percentile) by default — you need summary() for that.

Simulated PySpark

Value Counts and Distributions

For categorical columns, you want to know how values are distributed — how many students per major, how many in each year. These operations help you spot imbalances and understand your data's composition.

  • df.groupBy("major").count().show() — frequency count per category
  • df.select("year").distinct().count() — number of unique values in a column
Simulated PySpark

Sampling

When working with large datasets, you often want a quick peek at a subset rather than processing millions of rows. PySpark offers several ways to grab a sample:

  • df.sample(fraction=0.2) — random sample of approximately 20% of rows
  • df.take(5) — returns first 5 rows as a list of Row objects
  • df.limit(10).show() — limits to 10 rows and displays as a table
Simulated PySpark

Cross-tabulation

Cross-tabulation creates a pivot-table-like frequency matrix showing how two categorical variables relate to each other. It answers questions like "how many CS students are in year 3?" at a glance.

  • df.crosstab("major", "year") — produces a DataFrame with one row per value of the first column, and one column per value of the second
