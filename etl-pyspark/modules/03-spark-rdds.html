In Module 2, we saw how Spark keeps data in memory and uses lazy evaluation with DAGs. Now we'll look at the data structure that makes this possible: the RDD.

What is an RDD?

RDD stands for Resilient Distributed Dataset. Each word captures a key property of the abstraction:

  • Resilient — If a partition is lost (say a node crashes), Spark can reconstruct it by replaying the sequence of transformations that produced it. This chain of operations is called the lineage.
  • Distributed — The data is split into partitions spread across the nodes of a cluster. Each node works on its own slice in parallel.
  • Dataset — At the end of the day, it is simply a collection of elements: rows, tuples, numbers, strings, or any Python object.

One more property is critical: RDDs are immutable. You never modify an RDD in place. Every transformation (like map or filter) produces a brand-new RDD, leaving the original unchanged. This immutability is what makes lineage-based recovery possible: Spark can always rebuild from the original source by re-applying the recorded steps.

Think of an RDD like a read-only list that lives across multiple machines. You describe what you want to do to it, and Spark figures out how to do it efficiently across the cluster.

Transformations vs Actions

Every operation on an RDD falls into one of two categories:

Transformations (lazy)

Transformations describe a computation but do not execute it immediately. They return a new RDD and are only computed when an action triggers them.

  • map(func) — apply a function to each element
  • filter(func) — keep elements where the function returns True
  • flatMap(func) — like map, but each input can produce zero or more outputs
  • reduceByKey(func) — merge values for each key using a reduce function
  • groupByKey() — group values by key into iterables

Actions (trigger execution)

Actions force Spark to execute the chain of transformations and return a result to the driver program.

  • collect() — return all elements as a Python list
  • count() — return the number of elements
  • reduce(func) — aggregate all elements using a function
  • take(n) — return the first n elements
  • first() — return the first element (same as take(1)[0])
