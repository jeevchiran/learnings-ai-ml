In Module 6, we wrote UDFs to apply custom logic to our data. Now we face a real-world challenge: our dataset has problems. Before any analysis, we need to clean it.

Introducing the Messy Data

Our file grades_messy.csv has a range of quality issues that are typical of real-world datasets. Before we can do any meaningful analysis, we need to identify and fix these problems:

  • Null values — missing student IDs or grades
  • Duplicate rows — the same enrollment appearing more than once
  • Inconsistent casing — "Fall 2024", "SPRING 2025", "spring 2025"
  • Extra whitespace — " Data Structures " with leading/trailing spaces
  • Invalid grades — negative values or scores above 100
