Joins

Joins combine rows from two DataFrames based on a shared key column. PySpark supports several join types, each returning a different subset of the combined data:

  • Inner join — returns only rows where the key exists in both DataFrames.
  • Left join — returns all rows from the left DataFrame; unmatched rows from the right side get nulls.
  • Right join — returns all rows from the right DataFrame; unmatched rows from the left side get nulls.
  • Full outer join — returns all rows from both DataFrames; unmatched sides get nulls.
  • Anti join — returns rows from the left DataFrame that have NO match in the right DataFrame.

The syntax is: students_df.join(enrollments_df, "student_id", "inner")

Window Functions

Window functions compute values across a set of rows related to the current row — without collapsing groups into single rows like aggregation does. You define a window specification that controls partitioning and ordering, then apply ranking or analytic functions over it.

Key functions:

  • row_number() — assigns a unique sequential number within each partition.
  • rank() — assigns rank with gaps for ties (1, 2, 2, 4).
  • dense_rank() — assigns rank without gaps (1, 2, 2, 3).
  • lag(col, n) — accesses a value from a previous row within the partition.
  • lead(col, n) — accesses a value from a subsequent row within the partition.
