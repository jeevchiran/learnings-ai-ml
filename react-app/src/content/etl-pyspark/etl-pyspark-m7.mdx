## Introducing the Messy Data

Our file `grades_messy.csv` has a range of quality issues typical of real-world datasets:

- **Null values** — missing student IDs or grades
- **Duplicate rows** — the same enrollment appearing more than once
- **Inconsistent casing** — "Fall 2024", "SPRING 2025", "spring 2025"
- **Extra whitespace** — " Data Structures " with leading/trailing spaces
- **Invalid grades** — negative values or scores above 100

## Handling Nulls

PySpark gives you three strategies depending on whether you want to find, drop, or fill nulls.

### Finding Nulls

<CodeBlock language="python">
{`from pyspark.sql.functions import col

# Find rows where student_id is null
messy_df.filter(col("student_id").isNull()).show()`}
</CodeBlock>

### Dropping Nulls

<CodeBlock language="python">
{`# Drop rows with ANY null value
no_nulls = messy_df.dropna()

# Drop only where student_id is null
valid_students = messy_df.dropna(subset=["student_id"])`}
</CodeBlock>

### Filling Nulls

<CodeBlock language="python">
{`# Fill null grades with 0
filled_df = messy_df.fillna({"grade": 0})`}
</CodeBlock>

## Removing Duplicates

<CodeBlock language="python">

## Fixing Inconsistent Text

PySpark provides `trim()`, `lower()`, and `initcap()` to normalize text:

<CodeBlock language="python">
{`from pyspark.sql.functions import trim, lower, initcap

fixed_text = messy_df \
    .withColumn("course_name", trim(initcap(col("course_name")))) \
    .withColumn("semester", initcap(lower(col("semester"))))

# " Data Structures " → "Data Structures"
# "SPRING 2025" → "Spring 2025"
# "fall 2024" → "Fall 2024"`}
</CodeBlock>

## Validating Ranges

<CodeBlock language="python">

## Complete Cleaning Pipeline

In practice, chain all cleaning steps together using PySpark's fluent API:

<CodeBlock language="python">
{`from pyspark.sql.functions import col, trim, lower, initcap

cleaned_df = (
    messy_df
    .dropDuplicates(["enrollment_id"])
    .filter(col("student_id").isNotNull())
    .filter((col("grade") >= 0) & (col("grade") <= 100))
    .withColumn("course_name", trim(initcap(col("course_name"))))
    .withColumn("semester", initcap(lower(col("semester"))))
)

print(f"Cleaned: {cleaned_df.count()} rows (from original {messy_df.count()})") 
# Cleaned: 5 rows (from original 10)`}
</CodeBlock>

<ConceptBox title="Pipeline Order Matters">
The order of cleaning steps affects performance. Put the cheapest, highest-selectivity filters first:

1. `dropDuplicates` early — reduces rows before expensive string operations
2. `filter(isNotNull)` — eliminates rows that would fail later operations
3. `filter(range)` — cheap arithmetic filter
4. String transformations last — applied only to surviving rows

Spark's Catalyst optimizer can reorder some filters automatically, but explicit ordering makes intent clear and aids debugging.
</ConceptBox>
