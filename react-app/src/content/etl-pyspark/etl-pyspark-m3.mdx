In Module 2, we saw how Spark keeps data in memory and uses lazy evaluation with DAGs. Now we'll look at the data structure that makes this possible: the RDD.

## What is an RDD?

RDD stands for **Resilient Distributed Dataset**. Each word captures a key property of the abstraction:

- **Resilient** — If a partition is lost (say a node crashes), Spark can reconstruct it by replaying the sequence of transformations that produced it. This chain of operations is called the *lineage*.
- **Distributed** — The data is split into partitions spread across the nodes of a cluster. Each node works on its own slice in parallel.
- **Dataset** — At the end of the day, it is simply a collection of elements: rows, tuples, numbers, strings, or any Python object.

One more property is critical: RDDs are **immutable**. You never modify an RDD in place. Every transformation (like `map` or `filter`) produces a brand-new RDD, leaving the original unchanged. This immutability is what makes lineage-based recovery possible: Spark can always rebuild from the original source by re-applying the recorded steps.

Think of an RDD like a read-only list that lives across multiple machines. You describe what you want to do to it, and Spark figures out how to do it efficiently across the cluster.

## Creating RDDs

There are two primary ways to create an RDD in PySpark:

### From a Python collection

Use `sc.parallelize()` to distribute a local list across the cluster:

```python
students = [
    (1001, 'Alice Chen', 'Computer Science', 3, 3.8),
    (1002, 'Bob Martinez', 'Data Science', 2, 3.5),
    (1003, 'Carol Johnson', 'Mathematics', 4, 3.9),
    (1004, 'David Kim', 'Computer Science', 1, 3.2),
    (1005, 'Eva Patel', 'Data Science', 3, 3.7),
]

rdd = sc.parallelize(students, numSlices=4)
print(rdd)
print(f"Number of partitions: {rdd.getNumPartitions()}")
print(f"First 3 elements: {rdd.take(3)}")
```

### From an external file

Use `sc.textFile()` to read a file (local or HDFS) into an RDD of strings, one per line:

```python
lines_rdd = sc.textFile("students.csv")
print("First 3 lines:")
for line in lines_rdd.take(3):
    print(line)
print(f"\nTotal lines: {lines_rdd.count()} (1 header + 8 students)")
```

## Transformations vs Actions

Every operation on an RDD falls into one of two categories:

### Transformations (lazy)

Transformations describe a computation but do not execute it immediately. They return a new RDD and are only computed when an action triggers them.

- `map(func)` — apply a function to each element
- `filter(func)` — keep elements where the function returns True
- `flatMap(func)` — like map, but each input can produce zero or more outputs
- `reduceByKey(func)` — merge values for each key using a reduce function
- `groupByKey()` — group values by key into iterables

### Actions (trigger execution)

Actions force Spark to execute the chain of transformations and return a result to the driver program.

- `collect()` — return all elements as a Python list
- `count()` — return the number of elements
- `reduce(func)` — aggregate all elements using a function
- `take(n)` — return the first n elements
- `first()` — return the first element (same as take(1)[0])

### Putting it together

```python
students = [
    (1001, 'Alice Chen', 'Computer Science', 3, 3.8),
    (1002, 'Bob Martinez', 'Data Science', 2, 3.5),
    (1003, 'Carol Johnson', 'Mathematics', 4, 3.9),
    (1004, 'David Kim', 'Computer Science', 1, 3.2),
    (1005, 'Eva Patel', 'Data Science', 3, 3.7),
    (1006, 'Frank Wilson', 'Statistics', 2, 3.4),
    (1007, 'Grace Lee', 'Computer Science', 4, 3.6),
    (1008, 'Henry Brown', 'Mathematics', 1, 2.9),
]

rdd = sc.parallelize(students)

# Transformation: extract just names
names_rdd = rdd.map(lambda x: x[1])

# Action: collect results
names = names_rdd.collect()
print(f"Names: {names}")
```

```python
rdd = sc.parallelize(students)

# Transformation: filter by GPA > 3.5
high_gpa_rdd = rdd.filter(lambda x: x[4] > 3.5)

# Action: collect and display
results = high_gpa_rdd.collect()
print("Students with GPA > 3.5:")
for s in results:
    print(f"  {s[1]} ({s[4]})")
```

```python
rdd = sc.parallelize(students)

# Map each student to (major, 1) then reduce by key to count
major_counts = rdd.map(lambda x: (x[2], 1)) \
                  .reduceByKey(lambda a, b: a + b) \
                  .collect()

print("Students per major:")
for major, count in major_counts:
    print(f"  {major}: {count}")
```

## Live Demo: Python Analogues

RDD operations like `map`, `filter`, and grouping have direct equivalents in plain Python.

```python
# Python list operations mirror RDD transformations
students = [
    (1001, 'Alice Chen', 'Computer Science', 3, 3.8),
    (1002, 'Bob Martinez', 'Data Science', 2, 3.5),
    (1003, 'Carol Johnson', 'Mathematics', 4, 3.9),
    (1004, 'David Kim', 'Computer Science', 1, 3.2),
    (1005, 'Eva Patel', 'Data Science', 3, 3.7),
    (1006, 'Frank Wilson', 'Statistics', 2, 3.4),
    (1007, 'Grace Lee', 'Computer Science', 4, 3.6),
    (1008, 'Henry Brown', 'Mathematics', 1, 2.9),
]

# map: extract names (like rdd.map(lambda x: x[1]))
names = list(map(lambda s: s[1], students))
print("Names:", names)

# filter: GPA > 3.5 (like rdd.filter(lambda x: x[4] > 3.5))
high_gpa = list(filter(lambda s: s[4] > 3.5, students))
print(f"\nStudents with GPA > 3.5: {len(high_gpa)}")
for s in high_gpa:
    print(f"  {s[1]} ({s[4]})")

# reduce: group by major and count
from collections import Counter
major_counts = Counter(s[2] for s in students)
print(f"\nStudents per major:")
for major, count in major_counts.items():
    print(f"  {major}: {count}")
```

**Apply — Problem 2**

Using `map` and `filter` on an RDD of student tuples, extract the names of students with GPA > 3.5.

```python
data = [
    (1001, 'Alice Chen', 3.8),
    (1002, 'Bob Martinez', 3.5),
    (1003, 'Carol Johnson', 3.9),
    (1004, 'David Kim', 3.2),
    (1005, 'Eva Patel', 3.7),
]

rdd = sc.parallelize(data)

# Filter students with GPA > 3.5, then map to extract names
result = rdd.filter(lambda x: x[2] > 3.5) \
            .map(lambda x: x[1]) \
            .collect()

print(f"High-GPA student names: {result}")
```

**Stretch — Problem 3**

Use `reduceByKey` to compute the average GPA per major from an RDD of `(major, gpa)` pairs. Map each pair to `(major, (gpa, 1))`, reduce to sum GPAs and counts, then divide.

```python
data = [
    ('Computer Science', 3.8),
    ('Data Science', 3.5),
    ('Mathematics', 3.9),
    ('Computer Science', 3.2),
    ('Data Science', 3.7),
    ('Statistics', 3.4),
    ('Computer Science', 3.6),
    ('Mathematics', 2.9),
]

rdd = sc.parallelize(data)

# Step 1: Map to (major, (gpa, 1))
# Step 2: reduceByKey to sum GPAs and counts
# Step 3: Map to compute average
avg_gpa = rdd.map(lambda x: (x[0], (x[1], 1))) \
             .reduceByKey(lambda a, b: (a[0] + b[0], a[1] + b[1])) \
             .map(lambda x: (x[0], round(x[1][0] / x[1][1], 2))) \
             .collect()

print("Average GPA per major:")
for major, avg in avg_gpa:
    print(f"  {major}: {avg:.2f}")
```
