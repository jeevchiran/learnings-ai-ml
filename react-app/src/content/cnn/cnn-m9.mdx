# Module 9: Visualising & Interpreting CNN Activations

<Bridge>
**From Module 8:** transfer learning works because pretrained features generalise. But *which* features, and *where* in the image do they activate? Treating a CNN as a black box is both intellectually unsatisfying and practically risky — a model that achieves 95% accuracy by detecting a watermark instead of the actual object is worthless in deployment. This module covers the tools for looking inside.
</Bridge>

## Why visualise?

1. **Debugging:** a model achieving 99% train / 60% val accuracy is overfitting to spurious correlations. Activation maps reveal *what* it's attending to.
2. **Fairness audits:** does the model detect face colour or the actual skin condition?
3. **Scientific insight:** what has the network actually learned about its domain?
4. **Failure analysis:** which inputs trip the model? What does it see when it's wrong?

## Method 1: Intermediate activation maps

Extract the feature map at layer $l$ for an input $\mathbf{x}$:

```python
import torch, torchvision.models as models
import torchvision.transforms as T
from PIL import Image

model = models.resnet50(weights='IMAGENET1K_V1').eval()

# Register a hook to capture the output of layer3
activations = {}
def hook(module, input, output):
    activations['layer3'] = output.detach()

model.layer3.register_forward_hook(hook)

transform = T.Compose([T.Resize(224), T.CenterCrop(224), T.ToTensor(),
                        T.Normalize([0.485,0.456,0.406],[0.229,0.224,0.225])])
img = transform(Image.open('cat.jpg')).unsqueeze(0)
model(img)  # forward pass fires hook

feat = activations['layer3'][0]  # [C, H, W]
# Each channel is a different learned feature detector
```

Visualise each channel as a heatmap. Early channels (conv1) show Gabor-like patterns; deeper channels respond to eyes, fur texture, ears, etc.

## Method 2: GradCAM

**Gradient-weighted Class Activation Mapping** (Selvaraju et al., 2017) answers: "which spatial regions most influenced the class score?"

Algorithm:
1. Forward pass: compute class logit $y^c$ for the target class $c$
2. Backward pass: compute gradient of $y^c$ w.r.t. last conv layer $\mathbf{A}^k$
3. Pool: $\alpha_k^c = \frac{1}{Z} \sum_{i,j} \frac{\partial y^c}{\partial A^k_{ij}}$ (global average of gradient per channel)
4. Weighted sum: $L^c_{GradCAM} = \text{ReLU}\left(\sum_k \alpha_k^c \mathbf{A}^k\right)$

The ReLU keeps only locations with positive influence on the target class.

```python
model.zero_grad()
logits = model(img)
score = logits[0, target_class]
score.backward()

# Gradients at layer3
grads = model.layer3[5].conv3.weight.grad  # simplified — use hooks in practice
gap_weights = grads.mean(dim=[1, 2, 3])    # global average per channel

# Feature maps from layer3
A = activations['layer3'][0]  # [C, H, W]
cam = torch.zeros(A.shape[1:])
for k, w in enumerate(gap_weights):
    cam += w * A[k]
cam = torch.relu(cam)
```

Upsample the resulting $H_{conv} \times W_{conv}$ map to the input resolution and overlay on the original image.

## Method 3: Saliency maps (gradient-based)

Compute the gradient of the class score directly w.r.t. the input pixels:

$$\mathbf{S} = \left|\frac{\partial y^c}{\partial \mathbf{x}}\right|$$

Pixels where a small change strongly affects the score are "important." Saliency maps are noisy but computationally cheap (one backward pass). Integrated Gradients (Sundararajan et al., 2017) smooths this by integrating the gradient along a path from a baseline (black image) to the input.

## Method 4: Filter visualisation

Ask: "what input maximises the activation of filter $k$ at layer $l$?"

$$\mathbf{x}^* = \arg\max_{\mathbf{x}} A^k_l(\mathbf{x}) - \lambda \|\mathbf{x}\|_2^2$$

Optimise via gradient ascent on the input (starting from Gaussian noise):

```python
img_var = torch.randn(1, 3, 224, 224, requires_grad=True)
optimizer = torch.optim.Adam([img_var], lr=0.1)

for i in range(200):
    optimizer.zero_grad()
    act = model.layer2(model.layer1(model.maxpool(model.relu(model.bn1(model.conv1(img_var))))))
    loss = -act[0, target_filter].mean()  # maximise filter activation
    loss.backward()
    optimizer.step()
```

Early layer filters learn Gabor-like edge detectors. Deeper filters produce surreal textures that resemble the patterns found in Deep Dream.

<ConceptBox title="The limits of post-hoc explanations">
GradCAM shows where the model looks but not *why* it uses those features. A model that attends correctly to a dog's face might still be responding to fur texture rather than the face structure. Verification requires adversarial testing (occlude the attended region, observe accuracy drop) rather than visual inspection alone.
</ConceptBox>

## Evaluation: does the heatmap align with the object?

**Pointing game accuracy:** for each test image, check if the max-activation pixel of the GradCAM map falls within the ground-truth bounding box.

State-of-the-art GradCAM variants achieve ~70–80% pointing game accuracy on PASCAL VOC — not perfect, but far better than random (30–40%).

<DerivationSteps>
<DerivationStep number={1} title="GradCAM weights — derivation">

Let $y^c$ be the logit for class $c$ and $A^k_{ij}$ be the $(i,j)$ element of the $k$-th feature map in the last conv layer.

The neuron importance weight for channel $k$:

$$\alpha_k^c = \underbrace{\frac{1}{Z} \sum_i \sum_j}_{\text{global average pool}} \frac{\partial y^c}{\partial A^k_{ij}}$$

This is the gradient information pooled over all spatial positions. It measures how much, on average, the class score changes when channel $k$'s entire spatial activation changes.

The class activation map:

$$L^c_{GradCAM} = \text{ReLU}\left(\sum_k \alpha_k^c \cdot \mathbf{A}^k\right)$$

The ReLU discards channels that *suppress* the target class score (negative $\alpha_k^c$), keeping only those that *excite* it.

</DerivationStep>

<DerivationStep number={2} title="Class Activation Maps (CAM) — predecessor to GradCAM">

In a network using **Global Average Pooling** before the FC layer, the penultimate representation is:

$$F_k = \frac{1}{Z} \sum_{ij} A^k_{ij}$$

The class score: $y^c = \sum_k w^c_k F_k$ (no bias, for simplicity).

Substituting:

$$y^c = \sum_k w^c_k \frac{1}{Z} \sum_{ij} A^k_{ij} = \frac{1}{Z} \sum_{ij} \underbrace{\sum_k w^c_k A^k_{ij}}_{M^c(i,j)}$$

$M^c(i,j)$ is the **Class Activation Map** at position $(i,j)$ — the direct, no-backprop version of GradCAM that only works with GAP architectures. GradCAM generalises CAM to arbitrary architectures by replacing $w^c_k$ with the gradient pooled over positions.

</DerivationStep>

<DerivationStep number={3} title="Integrated Gradients — path integral formulation">

Vanilla saliency $\partial y^c / \partial x_i$ is noisy because gradient magnitudes fluctuate near saturation regions. Integrated Gradients (IG) fixes this by averaging gradients along a path from baseline $\mathbf{x}'$ (black image) to input $\mathbf{x}$:

$$\text{IG}_i(\mathbf{x}) = (x_i - x'_i) \cdot \int_0^1 \frac{\partial F(\mathbf{x}' + \alpha(\mathbf{x} - \mathbf{x}'))}{\partial x_i} \, d\alpha$$

In practice, approximate with $m$ steps:
$$\text{IG}_i \approx (x_i - x'_i) \cdot \frac{1}{m} \sum_{k=1}^m \frac{\partial F(\mathbf{x}' + \frac{k}{m}(\mathbf{x}-\mathbf{x}'))}{\partial x_i}$$

IG satisfies the **completeness axiom**: $\sum_i \text{IG}_i = F(\mathbf{x}) - F(\mathbf{x}')$. The attributions sum to the difference in output, so no "attribution" is fabricated or lost.

</DerivationStep>
</DerivationSteps>
