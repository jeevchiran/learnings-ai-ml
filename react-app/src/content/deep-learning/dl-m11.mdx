## What is Overfitting?

Overfitting occurs when a model learns the **training data too well** — including its noise — and fails to generalize to unseen examples. Symptoms:

- Training loss decreases → validation loss increases (divergence)
- Training accuracy high → validation accuracy low (gap)
- Model output is highly sensitive to small input perturbations

<ConceptBox title="The Bias-Variance Tradeoff">
Prediction error = Bias² + Variance + Irreducible noise

- **High bias (underfitting):** Model too simple, cannot capture true pattern
- **High variance (overfitting):** Model too complex, fits noise as well as signal
- **Goal:** Find the sweet spot that minimizes total error on unseen data
</ConceptBox>

## Why Overfitting Happens

With enough parameters, a neural network can memorize any training set with loss → 0. Neural networks with millions of parameters are massively overparameterized relative to typical dataset sizes. Several forces keep them from overfitting in practice:

1. **Stochastic gradient noise** — mini-batches add noise that acts as implicit regularization
2. **Early stopping** — stop before the model starts memorizing
3. **Explicit regularization** — L2/dropout
4. **Data augmentation** — show more diverse training examples

## Early Stopping

Monitor validation loss at each epoch. Stop training when validation loss stops improving for $k$ consecutive epochs (patience $k$).

```python
class EarlyStopping:
    def __init__(self, patience=10, min_delta=1e-4):
        self.patience = patience
        self.min_delta = min_delta
        self.best_loss = float('inf')
        self.counter = 0

    def __call__(self, val_loss):
        if val_loss < self.best_loss - self.min_delta:
            self.best_loss = val_loss
            self.counter = 0
            return False   # continue training
        self.counter += 1
        return self.counter >= self.patience   # True = stop
```

## End-to-End PyTorch Training Loop

A complete, production-quality training loop for MNIST digit classification:

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms

# ── 1. Data ─────────────────────────────────────────────
transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.1307,), (0.3081,))  # MNIST mean/std
])

train_ds = datasets.MNIST('./data', train=True,  download=True, transform=transform)
val_ds   = datasets.MNIST('./data', train=False, download=True, transform=transform)

train_loader = DataLoader(train_ds, batch_size=256, shuffle=True,  num_workers=2)
val_loader   = DataLoader(val_ds,   batch_size=512, shuffle=False, num_workers=2)

# ── 2. Model ─────────────────────────────────────────────
class MNISTClassifier(nn.Module):
    def __init__(self, dropout=0.3):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(784, 512),
            nn.BatchNorm1d(512),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(512, 256),
            nn.BatchNorm1d(256),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(256, 10),
        )

    def forward(self, x):
        return self.net(x.view(x.size(0), -1))  # flatten: [B,1,28,28] → [B,784]

device = 'cuda' if torch.cuda.is_available() else 'cpu'
model  = MNISTClassifier(dropout=0.3).to(device)

# ── 3. Loss & Optimizer ──────────────────────────────────
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4)
scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=30)

# ── 4. Training Loop ─────────────────────────────────────
early_stop = EarlyStopping(patience=5)
best_val_acc = 0.0

def train_epoch(model, loader, optimizer, criterion, device):
    model.train()
    total_loss, correct, n = 0., 0, 0
    for X, y in loader:
        X, y = X.to(device), y.to(device)
        optimizer.zero_grad()
        logits = model(X)
        loss   = criterion(logits, y)
        loss.backward()
        optimizer.step()
        total_loss += loss.item() * len(y)
        correct    += (logits.argmax(1) == y).sum().item()
        n          += len(y)
    return total_loss / n, correct / n

def eval_epoch(model, loader, criterion, device):
    model.eval()
    total_loss, correct, n = 0., 0, 0
    with torch.no_grad():
        for X, y in loader:
            X, y = X.to(device), y.to(device)
            logits = model(X)
            loss   = criterion(logits, y)
            total_loss += loss.item() * len(y)
            correct    += (logits.argmax(1) == y).sum().item()
            n          += len(y)
    return total_loss / n, correct / n

for epoch in range(50):
    train_loss, train_acc = train_epoch(model, train_loader, optimizer, criterion, device)
    val_loss,   val_acc   = eval_epoch(model, val_loader, criterion, device)
    scheduler.step()

    print(f'Epoch {epoch+1:3d} | '
          f'Train Loss: {train_loss:.4f}  Acc: {train_acc:.4f} | '
          f'Val Loss: {val_loss:.4f}  Acc: {val_acc:.4f}')

    if val_acc > best_val_acc:
        best_val_acc = val_acc
        torch.save(model.state_dict(), 'best_model.pt')

    if early_stop(val_loss):
        print(f'Early stop at epoch {epoch+1}')
        break

print(f'Best val accuracy: {best_val_acc:.4f}')
```

## Mathematics: Generalization Bounds

<DerivationSteps>

<DerivationStep number={1} title="Train-test gap and VC dimension">

For a hypothesis class $\mathcal{H}$ with VC dimension $d$, the generalization bound (with probability $1-\delta$):

$$J_{\text{test}}(\theta) \leq J_{\text{train}}(\theta) + \sqrt{\frac{d \cdot \log(2m/d) + \log(2/\delta)}{2m}}$$

where $m$ is the training set size. The gap scales as $O(\sqrt{d/m})$.

Neural networks have astronomically high VC dimension yet generalize well — this remains an open theoretical question ("double descent" phenomenon).

</DerivationStep>

<DerivationStep number={2} title="Early stopping as implicit regularization">

Early stopping at epoch $t^*$ corresponds to an $L2$ regularizer with $\lambda \propto 1/(\eta t^*)$:

$$\theta_{t^*} \approx \arg\min_\theta \left[ J(\theta) + \frac{1}{2\eta t^*} \|\theta - \theta_0\|^2 \right]$$

Stopping earlier (smaller $t^*$) → stronger regularization. This equivalence holds exactly for convex objectives with gradient descent.

</DerivationStep>

<DerivationStep number={3} title="Dropout as ensemble approximation">

With $n$ neurons and dropout rate $p$, we train a distribution over $2^n$ sub-networks. The exact ensemble prediction is:

$$p(\hat{y} \mid x) = \sum_{\mathbf{m}} p(\mathbf{m}) p(\hat{y} \mid x, \mathbf{m})$$

where $\mathbf{m}$ is the dropout mask. Computing this exactly is intractable. The standard inference approximation (no dropout, weights scaled by $(1-p)$) is a **geometric mean** approximation to this ensemble:

$$p_{\text{approx}}(\hat{y} \mid x) \approx \exp\!\left( \mathbb{E}_\mathbf{m} [\log p(\hat{y} \mid x, \mathbf{m})] \right)$$

This approximation is tight when $p$ is small and $n$ is large.

</DerivationStep>

<DerivationStep number={4} title="Putting it all together — the regularized training objective">

The full training objective with all techniques combined:

$$J_\text{full}(\theta) = \underbrace{\frac{1}{m}\sum_{i=1}^m \mathcal{L}(\hat{y}^{(i)}, y^{(i)})}_\text{task loss} + \underbrace{\frac{\lambda_2}{2m}\|\theta\|_2^2}_\text{L2} + \underbrace{\frac{\lambda_1}{m}\|\theta\|_1}_\text{L1}$$

Dropout and BatchNorm are not added to the objective — they modify the computation graph during training and have no explicit loss term.

Gradient of the full objective:
$$\nabla_\theta J_\text{full} = \nabla_\theta J + \frac{\lambda_2}{m}\theta + \frac{\lambda_1}{m}\text{sign}(\theta)$$

</DerivationStep>

</DerivationSteps>
