## What is a Multilayer Perceptron (MLP)?

A **Multilayer Perceptron** (MLP) is a feedforward neural network with at least one hidden layer between input and output. Each layer is a set of perceptrons whose outputs feed into the next layer.

<ConceptBox title="MLP Structure">
An MLP with $L$ layers has:
- **Layer 0 (input):** $n^{[0]}$ features $\mathbf{x} \in \mathbb{R}^{n^{[0]}}$
- **Layer $l$ (hidden, $1 \le l \le L-1$):** $n^{[l]}$ neurons computing $\mathbf{a}^{[l]} = g^{[l]}(W^{[l]}\mathbf{a}^{[l-1]} + \mathbf{b}^{[l]})$
- **Layer $L$ (output):** $n^{[L]}$ output neurons

Each weight matrix $W^{[l]} \in \mathbb{R}^{n^{[l]} \times n^{[l-1]}}$ connects layer $l-1$ to layer $l$.
</ConceptBox>

## Why Hidden Layers?

### Solving XOR

XOR is the canonical example. A two-hidden-unit network with step activations can solve it exactly:

- Hidden unit 1: $h_1 = \text{step}(x_1 + x_2 - 0.5)$ — "is at least one input on?"
- Hidden unit 2: $h_2 = \text{step}(x_1 + x_2 - 1.5)$ — "are both inputs on?"
- Output: $\hat{y} = \text{step}(h_1 - h_2 - 0.5)$

The hidden layer **transforms** the XOR problem into a linearly separable one in $(h_1, h_2)$ space.

### Universal Approximation Theorem

<ConceptBox title="Universal Approximation Theorem (Cybenko, 1989)">
A feedforward network with a single hidden layer containing a **finite** number of neurons with any non-constant, bounded, and continuous activation function can approximate any continuous function on a compact subset of $\mathbb{R}^n$ to arbitrary accuracy.
</ConceptBox>

**Important caveats:**
1. The theorem guarantees existence, not learnability — finding the weights may require exponentially many neurons.
2. **Depth is more efficient than width.** Deep networks can represent exponentially more functions than shallow networks of the same parameter count.
3. The theorem tells us MLPs are expressive. Training (backpropagation) tells us how to find the right weights.

## How Hidden Layers Work

### Feature Hierarchy

Each layer builds higher-level representations:
- **Layer 1:** Detects local patterns (edges, simple correlations)
- **Layer 2:** Combines local patterns into shapes
- **Layer 3+:** Combines shapes into concepts

This is not hand-engineered — the network learns what features are useful for the final task.

### Notation (used throughout the course)

For layer $l$ with weight matrix $W^{[l]}$ and bias vector $\mathbf{b}^{[l]}$:

$$\mathbf{z}^{[l]} = W^{[l]} \mathbf{a}^{[l-1]} + \mathbf{b}^{[l]} \quad \text{(pre-activation)}$$
$$\mathbf{a}^{[l]} = g^{[l]}(\mathbf{z}^{[l]}) \quad \text{(post-activation)}$$

where $\mathbf{a}^{[0]} = \mathbf{x}$ is the input.

**Dimensions:**
- $W^{[l]} \in \mathbb{R}^{n^{[l]} \times n^{[l-1]}}$
- $\mathbf{b}^{[l]} \in \mathbb{R}^{n^{[l]}}$
- $\mathbf{z}^{[l]}, \mathbf{a}^{[l]} \in \mathbb{R}^{n^{[l]}}$

## Depth vs Width

| Aspect | Shallow (1 wide hidden layer) | Deep (many narrow layers) |
|--------|-------------------------------|--------------------------|
| Expressiveness | Needs exponentially more neurons | Efficient with depth |
| Training difficulty | Generally easier | Vanishing gradients, needs careful init |
| Inductive bias | Less structured features | Hierarchical features |
| Practical performance | Works for simple tasks | Dominates complex tasks |

The consensus in practice: **more layers + nonlinear activations + proper training = better models.**

## Mathematics: L-Layer Forward Propagation

<DerivationSteps>

<DerivationStep number={1} title="Initialize with input">

Set $\mathbf{A}^{[0]} = \mathbf{X} \in \mathbb{R}^{n^{[0]} \times m}$ where $m$ is the batch size.

For a single sample: $\mathbf{a}^{[0]} = \mathbf{x} \in \mathbb{R}^{n^{[0]}}$.

</DerivationStep>

<DerivationStep number={2} title="Layer-by-layer computation">

For each layer $l = 1, 2, \ldots, L$:

$$\mathbf{Z}^{[l]} = W^{[l]} \mathbf{A}^{[l-1]} + \mathbf{b}^{[l]}$$
$$\mathbf{A}^{[l]} = g^{[l]}(\mathbf{Z}^{[l]})$$

The activation $g^{[l]}$ is applied **element-wise**. Different layers can have different activations (e.g., ReLU in hidden layers, sigmoid/softmax at output).

</DerivationStep>

<DerivationStep number={3} title="Output and prediction">

The network output is $\hat{\mathbf{Y}} = \mathbf{A}^{[L]}$.

For binary classification: $\hat{y} = \sigma(z^{[L]}) \in (0, 1)$

For multiclass classification: $\hat{\mathbf{y}} = \text{softmax}(\mathbf{z}^{[L]})$ where:

$$\text{softmax}(\mathbf{z})_k = \frac{e^{z_k}}{\sum_{j=1}^K e^{z_j}}$$

</DerivationStep>

<DerivationStep number={4} title="Why nonlinearity is essential">

If all $g^{[l]}$ were linear (or identity), then:

$$\mathbf{A}^{[L]} = W^{[L]} \cdots W^{[2]} W^{[1]} \mathbf{X} + \text{bias terms}$$

This collapses to a single linear transformation $\tilde{W}\mathbf{X} + \tilde{\mathbf{b}}$ — equivalent to a single-layer network regardless of depth. **Nonlinearity is what makes depth matter.**

</DerivationStep>

</DerivationSteps>

## Numerical Exercises: Counting MLP Parameters

The parameter count for a layer with $n_{in}$ inputs and $n_{out}$ outputs is:

$$\text{params}^{[l]} = n^{[l-1]} \times n^{[l]} + n^{[l]} = n^{[l]}(n^{[l-1]} + 1)$$

The $+1$ accounts for the bias term on each neuron.

### Problem 1 — MNIST Classifier

Architecture: $[784 \to 512 \to 256 \to 10]$

| Layer | $n_{in}$ | $n_{out}$ | Weights | Biases | Total |
|-------|---------|---------|---------|--------|-------|
| FC1 | 784 | 512 | 401,408 | 512 | **401,920** |
| FC2 | 512 | 256 | 131,072 | 256 | **131,328** |
| FC3 | 256 | 10 | 2,560 | 10 | **2,570** |
| **Total** | | | | | **535,818** |

Formula check: $784 \times 512 + 512 = 401{,}920$ ✓

### Problem 2 — CIFAR-10 MLP (why it explodes)

Input: $32 \times 32 \times 3 = 3{,}072$ pixels. Architecture: $[3072 \to 1024 \to 512 \to 10]$

| Layer | Params |
|-------|--------|
| FC1 ($3072 \to 1024$) | $3072 \times 1024 + 1024 = \mathbf{3{,}146{,}752}$ |
| FC2 ($1024 \to 512$) | $1024 \times 512 + 512 = \mathbf{524{,}800}$ |
| FC3 ($512 \to 10$) | $512 \times 10 + 10 = \mathbf{5{,}130}$ |
| **Total** | **3,676,682** |

A single FC layer from the image alone costs 3.1M parameters — and has zero spatial awareness. This is the central motivation for CNNs.

### Problem 3 — Depth vs Width Trade-off

Which has more parameters?
- Network A: $[100 \to 1000 \to 10]$ (wide, shallow)
- Network B: $[100 \to 100 \to 100 \to 100 \to 10]$ (narrow, deep)

Network A: $100 \times 1000 + 1000 + 1000 \times 10 + 10 = 111{,}010$

Network B: $100 \times 100 + 100 + 100 \times 100 + 100 + 100 \times 100 + 100 + 100 \times 10 + 10 = 31{,}310$

Network B uses **72% fewer parameters** but has 4 composition steps instead of 2. Deeper and narrower is often more parameter-efficient for the same expressive power.
