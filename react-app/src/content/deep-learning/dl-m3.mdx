## What is a Tensor?

A tensor in PyTorch is a generalization of scalars, vectors, and matrices to arbitrary dimensions. Critically, PyTorch tensors track every operation on them so that gradients can be computed automatically (autograd).

<ConceptBox title="Tensor Dimensions">
- **0-D (scalar):** a single number — `torch.tensor(3.14)`
- **1-D (vector):** shape `[n]` — `torch.tensor([1., 2., 3.])`
- **2-D (matrix):** shape `[m, n]` — weights of a single linear layer
- **3-D:** shape `[batch, seq, features]` — common in NLP
- **4-D:** shape `[batch, channels, height, width]` — common in vision
</ConceptBox>

## Why PyTorch?

1. **Dynamic computation graph (eager mode):** Operations execute immediately, making debugging trivial. You can use Python control flow (if/for) naturally.
2. **Autograd:** Every tensor with `requires_grad=True` tracks its computational history. Calling `.backward()` computes all gradients automatically.
3. **GPU acceleration:** Move tensors and models to GPU with `.to('cuda')` — same code, orders of magnitude faster.
4. **Ecosystem:** torchvision, torchaudio, HuggingFace, Lightning — the entire modern ML stack builds on PyTorch.

## How Tensors Work

### Creating Tensors

```python
import torch

# From data
x = torch.tensor([1.0, 2.0, 3.0])          # dtype inferred: float32

# Filled tensors
zeros = torch.zeros(3, 4)                   # shape [3, 4], all zeros
ones  = torch.ones(2, 3)
rand  = torch.rand(4, 4)                    # uniform [0, 1)
randn = torch.randn(4, 4)                   # standard normal N(0, 1)
eye   = torch.eye(3)                        # identity matrix

# Shape and dtype control
x = torch.zeros(2, 3, dtype=torch.float64)
print(x.shape)    # torch.Size([2, 3])
print(x.dtype)    # torch.float64
print(x.device)   # cpu
```

### Indexing and Slicing

```python
x = torch.rand(4, 5)

print(x[0])        # first row — shape [5]
print(x[:, 1])     # second column — shape [4]
print(x[1:3, 2:4]) # submatrix — shape [2, 2]

# Advanced indexing
rows = torch.tensor([0, 2])
print(x[rows])     # rows 0 and 2 — shape [2, 5]
```

### Broadcasting

PyTorch follows NumPy broadcasting rules. Two tensors are compatible if their shapes agree from the right, allowing size-1 dimensions to be expanded.

```python
a = torch.ones(3, 1)   # shape [3, 1]
b = torch.ones(1, 4)   # shape [1, 4]
c = a + b              # shape [3, 4] — broadcast
```

<ConceptBox title="Broadcasting Rule">
Align shapes from the right. If sizes differ, one must be 1 (and is expanded). Shapes `[3,1]` and `[1,4]` → result `[3,4]`. Shapes `[2,3]` and `[4,3]` are incompatible.
</ConceptBox>

### Core Operations

```python
x = torch.randn(3, 4)
y = torch.randn(4, 2)

# Matrix multiplication (the core of neural network layers)
z = torch.mm(x, y)         # [3, 4] @ [4, 2] = [3, 2]
z = x @ y                  # same, @ operator
z = torch.matmul(x, y)     # same, handles batching

# Batched matmul
A = torch.randn(8, 3, 4)   # batch of 8 matrices
B = torch.randn(8, 4, 2)
C = torch.bmm(A, B)        # [8, 3, 2]

# Element-wise
a = torch.randn(3)
b = torch.randn(3)
print(a + b)               # element-wise add
print(a * b)               # element-wise multiply (Hadamard ⊙)
print(a ** 2)              # element-wise square

# Reduction
print(x.sum())             # scalar
print(x.sum(dim=0))        # sum along rows, shape [4]
print(x.mean(dim=1))       # mean along cols, shape [3]
print(x.max())             # global max

# Reshape
x = torch.randn(12)
x = x.view(3, 4)           # reshape (must be contiguous)
x = x.reshape(2, 6)        # reshape (safer, handles non-contiguous)
x = x.unsqueeze(0)         # add dim at position 0: [1, 2, 6]
x = x.squeeze()            # remove all size-1 dims
```

### In-Place Operations

Operations ending in `_` modify the tensor in-place. **Be careful with in-place ops inside autograd graphs** — they can break gradient computation.

```python
x = torch.ones(3)
x.add_(1)      # in-place: x = x + 1
x.mul_(2)      # in-place: x = x * 2
# Same as: x = x + 1; x = x * 2  but avoids allocating new tensors
```

### GPU Transfer

```python
device = 'cuda' if torch.cuda.is_available() else 'cpu'

x = torch.randn(1000, 1000)
x = x.to(device)           # move to GPU (no-op if already there)

# All subsequent operations on x happen on GPU
y = x @ x.T                # GPU matmul
```

## Mathematics: Forward Propagation — Matrix Perspective

<DerivationSteps>

<DerivationStep number={1} title="Single sample vs batch">

**Single sample** $\mathbf{x} \in \mathbb{R}^{n}$:
$$\mathbf{z}^{[l]} = W^{[l]} \mathbf{x} + \mathbf{b}^{[l]}$$

**Batch of $m$ samples** $X \in \mathbb{R}^{n \times m}$ (columns are samples):
$$Z^{[l]} = W^{[l]} A^{[l-1]} + \mathbf{b}^{[l]}$$

where $\mathbf{b}^{[l]}$ is broadcast across all $m$ columns. This is a **single matrix multiply** — the GPU executes it in parallel across all samples.

In PyTorch:
```python
Z = A_prev @ W.T + b   # [m, n_out]  (convention: rows=samples)
# or equivalently:
Z = torch.addmm(b, A_prev, W.T)
```

</DerivationStep>

<DerivationStep number={2} title="Why float32?">

Default dtype is `torch.float32` (32-bit floating point). This trades off:

| dtype | bits | range | use case |
|-------|------|-------|---------|
| float64 | 64 | $\pm 10^{308}$ | scientific computing, when precision matters |
| float32 | 32 | $\pm 10^{38}$ | default DL training |
| float16 / bfloat16 | 16 | limited | mixed-precision training (faster, less memory) |

For gradient computations: float32 provides enough precision that numerical errors don't affect convergence in practice.

</DerivationStep>

<DerivationStep number={3} title="Computational complexity">

A single linear layer $\mathbf{z} = W\mathbf{a} + \mathbf{b}$ with $n_{in}$ inputs and $n_{out}$ outputs requires:

$$\text{FLOPs} = 2 \cdot n_{in} \cdot n_{out}$$

per sample (one multiply + one add per weight). For a batch of $m$:

$$\text{FLOPs per layer} = 2 \cdot m \cdot n_{in} \cdot n_{out}$$

GPUs execute thousands of such multiplications in parallel via SIMD instructions — this is why GPUs dominate deep learning.

</DerivationStep>

</DerivationSteps>
