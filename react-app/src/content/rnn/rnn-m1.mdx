# Module 1: Introduction to Sequential Data

<Bridge>
**Coming from Convolutional Neural Networks:** you can classify images using spatial hierarchies of features. But images are *static* — the 3×3 grid at position (5,5) carries no time-ordering information. Now consider a sentence, a stock price curve, or a speech signal. The same values in a different order mean something entirely different. This track is about models that process data *in order* — Recurrent Neural Networks.
</Bridge>

## What is sequential data?

Sequential data is any data where **position carries meaning**. Examples:

| Domain | Sequence | What order encodes |
|--------|----------|-------------------|
| Language | "The cat sat on the mat" | Syntax, semantics, grammar |
| Finance | [100, 103, 97, 110, 115] | Trend, momentum, volatility |
| Speech | Audio samples at 16 kHz | Phonemes, prosody, words |
| Video | Frame 1 → Frame 2 → … | Motion, causality, events |
| DNA | A-T-G-C-A-… | Protein encoding, gene function |
| Music | Note sequence | Melody, rhythm, harmony |

In every case, shuffling the sequence produces something meaningless or wrong.

## Why standard networks fail on sequences

A feedforward network (MLP or CNN) takes a **fixed-size input** and produces an output. Feed it a sentence of 12 words and it works. Feed it a sentence of 19 words — it can't, without changes. Feed it the same 12 words in a different order — it produces the same output, because the architecture is **permutation-invariant**.

This is the core problem. Sequential data has two properties that standard networks ignore:

**1. Variable length** — sequences have no fixed size. "OK" and "To be or not to be, that is the question" are both valid English sentences, but one has 2 tokens and the other has 13.

**2. Order dependency** — position matters. The word "not" completely changes the meaning of what follows. In a stock series, whether price 110 comes *before* or *after* 90 determines whether you're in a downtrend or an uptrend.

## What RNNs do differently

A Recurrent Neural Network maintains a **hidden state** $\mathbf{h}_t$ — a compact summary of everything seen up to time step $t$. At each step it reads two things: the new input $\mathbf{x}_t$ and the previous hidden state $\mathbf{h}_{t-1}$, producing a new hidden state:

$$\mathbf{h}_t = f(\mathbf{h}_{t-1},\, \mathbf{x}_t)$$

This loop gives the network **memory across time**. When processing word 8 in a sentence, $\mathbf{h}_7$ already contains a compressed representation of words 1–7. The RNN doesn't need to see them again.

<ConceptBox title="Hidden state as memory">
The hidden state $\mathbf{h}_t$ is a fixed-size vector (say, 128 floats) that summarises the entire input history $x_1, x_2, \ldots, x_t$. Think of it as a running note that the network keeps rewriting as it reads the sequence. The network learns *what to remember and what to forget* from data — not by hand-crafted rules.
</ConceptBox>

## The anatomy of a recurrence

Concretely, the standard RNN computes:

$$\mathbf{h}_t = \tanh\!\left(\mathbf{W}_h \mathbf{h}_{t-1} + \mathbf{W}_x \mathbf{x}_t + \mathbf{b}\right)$$

$$\hat{\mathbf{y}}_t = \mathbf{W}_y \mathbf{h}_t + \mathbf{b}_y$$

Three learned matrices — $\mathbf{W}_h$, $\mathbf{W}_x$, $\mathbf{W}_y$ — and two bias vectors. These are **shared across every time step** (crucial — we explore this in Module 7). The output $\hat{\mathbf{y}}_t$ is optional at intermediate steps; sometimes we only care about $\hat{\mathbf{y}}_T$, the output after reading the entire sequence.

## Folded vs unfolded view

The recurrence can be drawn two ways:

**Folded (compact):** one box with a self-loop arrow — the output feeds back as input at the next step.

**Unfolded (expanded):** the same box copied $T$ times in a line, one copy per time step. Arrows flow left to right, showing the hidden state passing from step to step.

Both views describe the same computation. The unfolded view is what you use when you think about training (backpropagation through time) and when you reason about what information is still reachable after $T$ steps.

<DerivationSteps>
<DerivationStep number={1} title="The recurrence relation — formal statement">

Let $T$ be the sequence length, $\mathbf{x}_t \in \mathbb{R}^d$ the input at step $t$, and $\mathbf{h}_t \in \mathbb{R}^n$ the hidden state.

The recurrence is:

$$\mathbf{h}_t = \tanh(\mathbf{W}_h \mathbf{h}_{t-1} + \mathbf{W}_x \mathbf{x}_t + \mathbf{b})$$

where $\mathbf{W}_h \in \mathbb{R}^{n \times n}$, $\mathbf{W}_x \in \mathbb{R}^{n \times d}$, $\mathbf{b} \in \mathbb{R}^n$.

The initial state $\mathbf{h}_0$ is typically set to $\mathbf{0}$, though it can also be learned.

</DerivationStep>
<DerivationStep number={2} title="Why tanh? The saturation trade-off">

tanh maps any real input to $(-1, 1)$:

$$\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}}$$

Its derivative is $\tanh'(z) = 1 - \tanh^2(z)$, which is 1 at $z=0$ and approaches 0 as $|z| \to \infty$.

This keeps the hidden state bounded — preventing it from growing without limit — but the near-zero derivative at saturation will cause problems during backpropagation (Module 9).

</DerivationStep>
<DerivationStep number={3} title="Parameter count of an RNN">

Given input dimension $d$, hidden size $n$, output size $k$:

| Matrix | Shape | Count |
|--------|-------|-------|
| $\mathbf{W}_x$ | $n \times d$ | $nd$ |
| $\mathbf{W}_h$ | $n \times n$ | $n^2$ |
| $\mathbf{b}$ | $n$ | $n$ |
| $\mathbf{W}_y$ | $k \times n$ | $kn$ |
| $\mathbf{b}_y$ | $k$ | $k$ |
| **Total** | | $nd + n^2 + n + kn + k$ |

For $d=50, n=128, k=10$: $50 \times 128 + 128^2 + 128 + 10 \times 128 + 10 = 24{,}074$ parameters — independent of sequence length $T$.

</DerivationStep>
</DerivationSteps>
