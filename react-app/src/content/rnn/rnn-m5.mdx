## Two views of the same network

An RNN can be drawn in two equivalent ways. The choice of view depends on what you want to reason about.

### Folded view тАФ the recurrence loop

The folded diagram shows a single cell with a **self-loop**: the output (hidden state $\mathbf{h}_t$) is fed back as input to the same cell at the next time step.

```
        тФМтФАтФАтФАтФАтФАтФР
x_t тФАтФАтЖТ тФВ     тФВтФАтФАтЖТ h_t
        тФВ RNN тФВ тЖЧ
        тФФтФАтФАтФАтФАтФАтФШ
           тЖС
         h_{t-1} (from previous step тАФ the loop)
```

This compact view emphasises that *there is only one set of weights* тАФ the same cell is reused every step. The loop is the defining structural feature.

### Unfolded view тАФ the computational graph

The unfolded diagram copies the cell $T$ times, one per time step, and shows the hidden state passing right:

```
hтВА тЖТ[RNN]тЖТ hтВБ тЖТ[RNN]тЖТ hтВВ тЖТ[RNN]тЖТ hтВГ тЖТ[RNN]тЖТ hтВД
      тЖС           тЖС           тЖС           тЖС
      xтВБ          xтВВ          xтВГ          xтВД
```

Each box uses the same weights. The unfolded view is what you use to reason about **backpropagation through time** (BPTT, Module 8) тАФ the gradient of the final loss must flow backward through each copy.

## What flows in and out

At each step $t$, three things participate:

| Signal | Symbol | Direction | Shape |
|--------|--------|-----------|-------|
| Input | $\mathbf{x}_t$ | Into cell from outside | $(d,)$ |
| Previous hidden state | $\mathbf{h}_{t-1}$ | Into cell from left | $(n,)$ |
| New hidden state | $\mathbf{h}_t$ | Out of cell to right | $(n,)$ |
| Output (optional) | $\hat{\mathbf{y}}_t$ | Out of cell downward | $(k,)$ |

The hidden state is the **only connection between time steps**. All information the network has from $x_1, \ldots, x_{t-1}$ is compressed into the $n$-dimensional vector $\mathbf{h}_{t-1}$. This is both the power of RNNs (compact, parameter-efficient) and their limitation (a bottleneck тАФ Module 9).

## Initial hidden state $\mathbf{h}_0$

The first cell has no predecessor. By convention, $\mathbf{h}_0 = \mathbf{0}$ (zero vector). Alternatively:
- $\mathbf{h}_0$ can be a **learned parameter** (trained by gradient descent)
- $\mathbf{h}_0$ can be **computed from a context vector** (encoder-decoder architecture)

In practice, zero initialisation works well for most tasks. Learned $\mathbf{h}_0$ adds $n$ parameters and sometimes helps on tasks where the network needs to "start" in a particular configuration.

## Reading the output

Where does the final prediction come from? There are two conventions:

**Last hidden state** тАФ use $\mathbf{h}_T$ (after reading all inputs) to make a single prediction. Used for many-to-one tasks like sentiment classification.

**All hidden states** тАФ use $\mathbf{h}_1, \ldots, \mathbf{h}_T$, one prediction per step. Used for many-to-many tasks like NER tagging.

```python
import torch.nn as nn

rnn = nn.RNN(input_size=50, hidden_size=128, batch_first=True)
# Input: (batch, seq_len, 50)
output, h_n = rnn(x)
# output: (batch, seq_len, 128) тАФ all hidden states h_1...h_T
# h_n:    (1, batch, 128)       тАФ final hidden state h_T
```

For classification, a linear layer maps $\mathbf{h}_T$ to class logits:

```python
classifier = nn.Linear(128, num_classes)
logits = classifier(h_n.squeeze(0))  # (batch, num_classes)
```

<ConceptBox title="Depth vs width in RNNs">
A single-layer RNN has one hidden state per step. A **stacked (deep) RNN** adds multiple layers: the output of RNN layer 1 becomes the input to RNN layer 2 at each time step. Depth adds capacity to model more complex functions of the history. In PyTorch, `nn.RNN(num_layers=2)` gives a 2-layer stacked RNN. The tradeoff: more parameters, slower training, potentially better performance.
</ConceptBox>

<DerivationSteps>
<DerivationStep number={1} title="Stacked RNN тАФ formal notation">

For a two-layer RNN with hidden sizes $n_1$ and $n_2$:

**Layer 1:** $\mathbf{h}_t^{(1)} = \tanh(\mathbf{W}_h^{(1)} \mathbf{h}_{t-1}^{(1)} + \mathbf{W}_x^{(1)} \mathbf{x}_t + \mathbf{b}^{(1)})$

**Layer 2:** $\mathbf{h}_t^{(2)} = \tanh(\mathbf{W}_h^{(2)} \mathbf{h}_{t-1}^{(2)} + \mathbf{W}_x^{(2)} \mathbf{h}_t^{(1)} + \mathbf{b}^{(2)})$

The output of the second layer is what we use for prediction: $\hat{\mathbf{y}}_t = \mathbf{W}_y \mathbf{h}_t^{(2)} + \mathbf{b}_y$.

Total parameters: $(n_1 \cdot d + n_1^2 + n_1) + (n_2 \cdot n_1 + n_2^2 + n_2) + (k \cdot n_2 + k)$.

</DerivationStep>
<DerivationStep number={2} title="Bidirectional RNN">

A **bidirectional RNN** runs two separate RNNs: one left-to-right (forward) and one right-to-left (backward). Their hidden states are concatenated at each step:

$$\overrightarrow{\mathbf{h}}_t = \text{RNN}_\text{fwd}(\mathbf{x}_t, \overrightarrow{\mathbf{h}}_{t-1})$$
$$\overleftarrow{\mathbf{h}}_t = \text{RNN}_\text{bwd}(\mathbf{x}_t, \overleftarrow{\mathbf{h}}_{t+1})$$
$$\mathbf{h}_t = [\overrightarrow{\mathbf{h}}_t \,;\, \overleftarrow{\mathbf{h}}_t]$$

This gives each position access to **both past and future context**. Useful for NER and POS tagging where the word "Washington" may be a person (Washington Carver) or a place (Washington D.C.) тАФ the following words disambiguate. **Cannot be used for generation** (you don't have future context at inference time).

</DerivationStep>
</DerivationSteps>
