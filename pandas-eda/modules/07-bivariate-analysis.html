Module 7: Bivariate Analysis

What, Why, and How

WHAT: Examining how two variables relate

Bivariate analysis looks at pairs of variables simultaneously — how one changes in relation to another. The core tools are:

  • Scatter plot — plots each observation as a point at (x, y). Reveals the direction, strength, and shape of any relationship, including non-linear curves or clusters.
  • Grouped box plot — shows a separate box plot for each category. Useful when one variable is categorical (e.g., payment_type) and the other is numeric (e.g., fare_amount).
  • Pearson correlation coefficient (r) — a single number summarizing the strength and direction of a linear relationship. Ranges from −1 (perfect negative) to +1 (perfect positive); 0 means no linear association.
  • Correlation matrix — extends Pearson r to every pair of numeric columns simultaneously, displayed as a grid.

WHY: Bivariate analysis reveals predictive signals and confounders

Knowing a column's distribution in isolation tells you nothing about whether it predicts something else. Bivariate analysis answers questions like:

  • Does trip_distance predict fare_amount? (Feature selection for a regression model)
  • Do card-paying passengers tip more than cash passengers? (Group comparison)
  • Are there variables that seem correlated only because both depend on a third variable? (Confounder detection)

In this dataset, fare_amount and trip_distance have a strong positive relationship for metered rides — but airport flat-rate fares (RatecodeID=2) and the $399 meter error row break that pattern. Bivariate analysis surfaces these complications before you build a model.

HOW: Scatter plots, df.corr(), and Pearson r

The Pearson correlation formula quantifies linear association:

r = \frac{\sum(x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum(x_i - \bar{x})^2 \cdot \sum(y_i - \bar{y})^2}}

The numerator is the covariance (how x and y deviate together); the denominator normalises by the product of standard deviations, bounding r between −1 and +1.

1
Scatter plot via pandas
df.plot.scatter(x='trip_distance', y='fare_amount', alpha=0.5) — each row becomes a point. alpha makes overlapping points visible by reducing opacity.
2
Correlation matrix via pandas
df[cols].corr() — returns a symmetric DataFrame where each cell is the Pearson r between the row column and column column. The diagonal is always 1.0.
3
Seaborn heatmap
sns.heatmap(corr_matrix, annot=True, cmap='coolwarm', center=0) — colours each cell by its r value: warm (red) for positive, cool (blue) for negative, white at zero.

Practice

Problem 1: Interpreting r = 0

Easy

What does a Pearson r of 0 mean?

r = 0 means there is no linear relationship between the two variables. When one variable increases, the other shows no systematic tendency to increase or decrease.

Critically, r = 0 does not mean the variables are unrelated. A perfect U-shaped (quadratic) relationship would also produce r ≈ 0 because positive and negative deviations cancel out in the formula. Always plot the scatter before concluding independence — a non-linear pattern can have r near zero while the variables are strongly associated.

Problem 2: Why is fare_amount vs trip_distance not r = 1.0?

Medium

NYC taxi fare is calculated from distance, yet fare_amount vs trip_distance does not achieve r = 1.0. Why not?

Several factors add variance beyond distance:

  • Flat-rate airport fares (RatecodeID=2, rows 6 and 18): fare is $45/$52 regardless of the exact mileage — a fixed amount that breaks the distance-to-price proportionality.
  • The $399 meter error (row 26): a 6.7-mile trip with a $399 fare is almost certainly a meter malfunction. This single point pulls the correlation away from the metered-trip pattern.
  • Negative fares (rows 7, 15): refunds or voids recorded as negative amounts have no valid relationship to distance.
  • Tolls and surcharges: these are included in total_amount but not in fare_amount, yet the flat surcharge amounts vary by route.
  • Time-based metering: slow traffic adds time-based charges, so two trips of the same distance can have different fares depending on traffic speed.

The lesson: even when you know a causal mechanism (distance drives fare), real-world data adds enough noise through exceptions and errors that r < 1.

Problem 3: Using the correlation matrix for feature selection

Hard

Describe how you would use the correlation matrix to do feature selection for a model predicting tip_amount.

A systematic approach:

  1. Look at the tip_amount row/column — check the absolute r value between tip_amount and every other numeric column. Features with |r| close to 0 contribute little predictive power via linear mechanisms; features with |r| > 0.4 are candidates to include.
  2. Check inter-feature correlations — if two candidate features are highly correlated with each other (e.g., r > 0.85), keeping both adds redundancy without information gain, and may cause multicollinearity in linear models. Drop the one with the lower correlation to the target.
  3. Interpret the findings — fare_amount and trip_distance likely have moderate positive r with tip_amount because higher fares typically receive larger tip amounts. passenger_count likely shows near-zero r, making it a weak linear predictor.
  4. Go beyond Pearson r — the correlation matrix only captures linear associations. A categorical feature like payment_type (card vs cash) could be a strong predictor even though it does not appear in a Pearson matrix. Use point-biserial correlation or group means to evaluate such features separately.
  5. Validate with domain knowledge — cash passengers cannot be tipped electronically (tip_amount is 0 for all cash rows in this dataset), so payment_type is a near-deterministic predictor that a correlation matrix would not flag as numeric.

Module 7 — Check Your Understanding

1. A Pearson r of −0.8 indicates:

A No relationship
B Weak negative relationship
C Strong negative linear relationship
D Perfect non-linear relationship
Correct. r = −0.8 is close to −1 (perfect negative linear), meaning as x increases, y decreases strongly and proportionally. By convention: |r| > 0.7 = strong, 0.4–0.7 = moderate, < 0.4 = weak. The negative sign indicates the direction (inverse), and 0.8 is in the strong range.
Not quite. The magnitude of r (0.8) tells you strength — it is close to 1, meaning strong. The negative sign tells you direction — as x increases, y tends to decrease. r = 0 would mean no relationship; |r| < 0.4 would mean weak. r only measures linear association, so "non-linear" is not what the value encodes.

2. What is the correlation of any variable with itself?

A 0
B −1
C 1.0
D Undefined
Correct. The Pearson formula for corr(x, x) simplifies to the covariance of x with itself divided by its own variance — which equals 1. This is why the diagonal of a correlation matrix is always 1.00, and is a quick sanity check that the computation is correct.
Not quite. Plug x into the Pearson formula as both variables: numerator becomes sum of (x−mean)² which equals the variance; denominator becomes sqrt(variance * variance) = variance. The ratio is 1. Every variable perfectly predicts itself — that is r = 1.

3. A scatter plot shows a curved (non-linear) pattern but Pearson r ≈ 0. This means:

A The variables are unrelated
B Pearson r only captures linear relationships; non-linear association exists
C The data has errors
D You need more data points
Correct. Pearson r is specifically a measure of linear association. A perfect U-shape (y = x²) produces r = 0 because the positive slope on one side cancels the negative slope on the other. This is why "always look at the scatter plot" is rule one — a low r does not mean independence.
Not quite. The scatter plot tells you a strong relationship exists — it is curved. Pearson r ≈ 0 just means no linear relationship. The two facts together reveal that the association is non-linear, not that it is absent. Spearman rank correlation or polynomial regression would capture this better.

4. In pandas, df.corr() computes:

A Covariance matrix
B Pearson correlation matrix for all numeric columns
C Spearman correlation only
D Chi-squared statistics
Correct. df.corr() defaults to method='pearson' and operates on all numeric columns, returning a symmetric n×n DataFrame. To get covariance use df.cov(); for Spearman use df.corr(method='spearman'). Chi-squared is for categorical independence tests, unrelated to corr().
Not quite. df.corr() is not the covariance matrix (that is df.cov()) — it normalises covariance by standard deviations, giving bounded r values. It defaults to Pearson; Spearman requires method='spearman'. Chi-squared is a completely separate test for categorical data.
