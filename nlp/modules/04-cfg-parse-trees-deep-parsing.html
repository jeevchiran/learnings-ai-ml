Module 4: Context-Free Grammars, Parse Trees & Deep Parsing

From Chunks to Full Structure

Module 3 gave us shallow parsing: a chunker labels spans like [NP the cat] and [NP the mat], but it does not tell you how those phrases relate to each other. Consider the sentence:

"The cat that sat on the mat looked up."

A chunker would identify several noun phrases and a verb phrase, but it cannot represent that "that sat on the mat" is a relative clause modifying "the cat", which in turn is the subject of "looked up". That nested relationship — an NP containing a clause containing its own NP — is invisible to a flat chunk sequence.

This is where deep parsing comes in. A full syntactic parse produces a constituency tree (also called a parse tree) that captures every level of embedding, showing how every word and phrase is grammatically governed by the structure above it.

Chomsky's central insight (1957) Natural language is recursive. An NP can contain a clause (relative clause), and that clause contains another NP, which might contain yet another clause. A finite grammar of rewriting rules generates an infinite set of grammatical sentences — the "infinite use of finite means." Context-Free Grammars (CFGs) are the formal system that captures this creativity.

Deep parsing has real-world impact wherever understanding grammatical structure matters: machine translation, question answering, grammar checking, information extraction, and semantic parsing all rely on knowing which constituent heads which clause.

The path from Module 3 to Module 4. Shallow parsing gave us labeled flat spans. This module formalises the recursive rule system (CFG) that generates those spans, shows how to represent them as trees, explains how parsers find those trees efficiently (CYK, Earley), and confronts the hard fact that most sentences are structurally ambiguous.

Context-Free Grammars — Formal Definition

A Context-Free Grammar is a 4-tuple:

G = (N,\, \Sigma,\, R,\, S)

The Four Components

  • N — a finite set of non-terminal symbols (syntactic categories): S, NP, VP, PP, DT, N, V, JJ, IN, …
  • Σ — a finite set of terminal symbols (the vocabulary of words): "the", "dog", "chased", …  Disjoint from N.
  • R — a finite set of production rules of the form A \to \alpha where A \in N and \alpha \in (N \cup \Sigma)^*. The right-hand side is any sequence of terminals and non-terminals (including the empty string ε).
  • S — the start symbol (by convention S for Sentence). Every grammatical sentence is derived from S.

Here is a small but workable English grammar:

S → NP VP NP → DT N | DT JJ N | NP PP | N VP → V NP | V PP | V NP PP | V PP → IN NP DT → "the" | "a" | "an" N → "dog" | "cat" | "bone" | "park" V → "chased" | "saw" | "ate" JJ → "big" | "small" | "lazy" IN → "in" | "with" | "on"

The vertical bar | is shorthand for alternative right-hand sides — NP → DT N | DT JJ N is simply two separate rules with the same left-hand side.

Derivations

A derivation is a sequence of rewriting steps starting from S and ending in a sequence of terminals (the sentence). At each step we pick one non-terminal and replace it with the right-hand side of one of its rules. A leftmost derivation always expands the leftmost non-terminal first.

Here is the complete leftmost derivation for "the dog chased a cat":

Apply S → NP VP

Parse Trees — Structure and Reading

A parse tree (constituency tree) is a visual representation of a derivation. It is a rooted, ordered tree where:

  • Internal nodes are labeled with non-terminal symbols (S, NP, VP, PP, …). Each internal node corresponds to one application of a production rule.
  • Leaf nodes are labeled with terminal words in left-to-right sentence order.
  • Each subtree rooted at an internal node represents a constituent — a grammatically coherent phrase.

Three key structural concepts:

Dominance

Node A dominates node B if there is a downward path from A to B. S dominates every node; NP dominates all words in that noun phrase.

Linear precedence

If the leftmost leaf under node A comes before the leftmost leaf under B in the sentence, then A precedes B. The tree encodes word order through left-to-right child ordering.

Headedness

Each phrase has a head — the word that determines the phrase's category. The head of an NP is typically the N; the head of a VP is the V. Heads govern agreement and subcategorization.

Structural Ambiguity

One of the most important (and humbling) facts about natural language syntax is that most sentences above a few words in length can be parsed in more than one way by a CFG. This is called structural ambiguity.

PP Attachment Ambiguity

The canonical example is prepositional phrase (PP) attachment. Consider:

"I saw the man with the telescope."

The PP "with the telescope" can attach to two different constituents:

  • VP attachment: I [VP saw the man [PP with the telescope]] — I used the telescope to see him.
  • NP attachment: I [VP saw [NP the man [PP with the telescope]]] — the man had/was holding the telescope.

Both are perfectly grammatical readings of the same word sequence. A CFG generates both parse trees with equal status. Choosing the right one requires world knowledge or statistical evidence from a corpus.

Garden Path Sentences

Garden path sentences are extreme cases where the preferred initial parse is wrong and the parser — human or machine — must backtrack:

"The horse raced past the barn fell."

Most readers initially parse "raced past the barn" as the main verb phrase, then hit "fell" and realise the sentence is actually passive: "The horse [that was] raced past the barn fell." The parser went down the garden path.

Ambiguity Grows Exponentially

For a sentence with n words, the number of distinct binary tree structures (and thus potential parse trees) follows the Catalan number sequence:

C_n = \frac{1}{n+1}\binom{2n}{n} = \frac{(2n)!}{(n+1)!\, n!}

For 3 attachment sites: C_3 = 5. For 10 attachment sites: C_{10} = 16{,}796. The combinatorial explosion is real — statistical parsers deal with it by assigning probabilities to rules and selecting the highest-probability parse tree.

How statistical parsers resolve ambiguity A Probabilistic CFG (PCFG) annotates each rule with a probability estimated from a treebank (a corpus of human-annotated parse trees, such as the Penn Treebank). The parser selects the most probable parse tree. "With the telescope" attaches to NP more often in the training corpus when the NP refers to a person, so the parser learns to prefer that attachment in those contexts.

Parsing Algorithms — Overview

Given a sentence and a CFG, a parser must find the parse tree (or all parse trees). Two classic families of algorithm dominate:

Top-Down Parsing (Earley Algorithm)

Start from S and repeatedly expand rules, predicting which non-terminals might cover upcoming input tokens. The Earley algorithm (1970) handles all CFGs in O(n^3) time (or O(n^2) for unambiguous grammars). It works by maintaining a chart of Earley items — partial rule applications with a dot showing how far they have progressed.

Bottom-Up Parsing (CYK Algorithm)

Start from individual words and combine adjacent constituents upward until a complete S is built. The Cocke-Younger-Kasami (CYK) algorithm requires the grammar to be in Chomsky Normal Form (CNF).

Chomsky Normal Form (CNF)

Every rule must be of exactly one of two forms:

  • Binary branching: A \to BC — a non-terminal rewrites to exactly two non-terminals.
  • Lexical: A \to a — a non-terminal rewrites to exactly one terminal.

Any CFG can be converted to CNF by binarising longer rules (introducing fresh non-terminals) and removing unit productions and ε-rules. See Practice Problem 3.

CYK fills a triangular table where cell [i,j] records all non-terminals that can derive the substring from position i to j. The sentence is grammatical if and only if S \in [1,n].

Time complexity:

O(n^3 \cdot |G|)

where n is the sentence length and |G| is the size of the grammar (number of rules).

Modern neural parsers Transition-based parsers (shift-reduce with a neural scorer) and graph-based parsers (biaffine attention over all pairs of words) have largely replaced pure CFG parsers in production NLP systems. They achieve near-human accuracy on Penn Treebank benchmarks (F1 > 95%) and run in near-linear time. However, they are best understood as learning to approximate the decisions that a rule-based CFG parser would make — the CFG framework remains the conceptual foundation.

Grammatical Agreement Checking

Agreement is the grammatical requirement that certain features (number, person, gender, case) must match between linked constituents. It is one of the most concrete applications of syntactic structure.

Subject-verb agreement

"The dog runs" (singular) vs. *"The dog run" (violation). The verb form must match the subject's number and person.

Number agreement

"The data are / show" — "data" is the Latin plural of "datum". Prescriptive grammar requires a plural verb; everyday usage accepts the singular.

Case agreement (pronouns)

English is mostly case-free, but pronouns retain it: "He/him", "she/her", "they/them". *"Him ran" violates case.

Rich agreement in other languages

German nouns have grammatical gender (der/die/das) and four cases. Latin verbs encode person, number, and tense in a single suffix. CFGs encode agreement via feature-annotated non-terminals.

CFGs handle agreement by subscripting non-terminals with agreement features. For English subject-verb agreement:

S → NP_sg VP_sg | NP_pl VP_pl NP_sg → DT N_sg NP_pl → DT N_pl VP_sg → V_sg NP VP_pl → V_pl NP N_sg → "dog" | "cat" N_pl → "dogs" | "cats" V_sg → "runs" | "chases" V_pl → "run" | "chase"

Under this grammar, "The dogs chases a cat" is ungrammatical because NP_pl requires VP_pl, but "chases" is a V_sg. Grammar checkers (Grammarly, LanguageTool) implement exactly this logic — they use a parser to find the subject-head NP, extract its number feature, and then check that the head verb's inflection matches.

Code: CFG Parsing with NLTK

spaCy does not expose a CFG interface directly — it uses a neural dependency parser. For explicit CFG work, NLTK provides a clean API.

Python · NLTK CFG 
import nltk

grammar = nltk.CFG.fromstring("""
  S  -> NP VP
  NP -> DT N | DT JJ N | N
  VP -> V NP | V PP
  PP -> IN NP
  DT -> 'the' | 'a'
  N  -> 'dog' | 'cat' | 'park'
  V  -> 'chased' | 'saw'
  JJ -> 'big' | 'lazy'
  IN -> 'in' | 'with'
""")

parser   = nltk.ChartParser(grammar)
sentence = ['the', 'dog', 'chased', 'a', 'big', 'cat']

for tree in parser.parse(sentence):
    print(tree)
    tree.pretty_print()

Problem 2: Center-Embedded Clauses and Catalan Numbers

Medium

Consider the sentence: "I saw the man the girl the dog bit scratched." (Three center-embedded relative clauses.)

(a) How many distinct NPs are in this sentence?

(b) Why is this sentence extremely difficult for humans to parse, despite being grammatically valid?

(c) The Catalan number C_n counts the number of distinct binary tree structures for a sequence of n+1 leaves. Write the formula for C_n and compute C_3.

(a) NPs: There are 4 NPs: "I", "the man", "the girl", "the dog". The sentence unpacks as: I saw [the man [the girl [the dog bit] scratched]].

(b) Psycholinguistic difficulty: Each relative clause is center-embedded — it interrupts the main clause rather than appearing at the end. The parser must hold the unresolved subject NPs in working memory ("the man", "the girl") while processing the inner clauses. By the time the verb "scratched" arrives, the parser must retroactively link it to "the girl" — but the intervening material has consumed the available working memory slots. Humans are typically limited to one level of center-embedding before comprehension collapses.

(c) Catalan number formula:

C_n = \frac{1}{n+1}\binom{2n}{n} = \frac{(2n)!}{(n+1)!\, n!}

For n = 3:

C_3 = \frac{1}{4}\binom{6}{3} = \frac{1}{4} \cdot 20 = 5

There are 5 distinct binary tree shapes for 4 leaves. This corresponds to 5 different ways to bracket the string ((AB)C)D, (A(BC))D, … All 5 might represent valid parse trees for an ambiguous 4-word phrase.

Problem 3: CNF Conversion

Hard

Convert the following rule to Chomsky Normal Form (CNF):

VP → V NP PP NP

CNF requires all rules to be either A \to BC or A \to a. Show all intermediate steps and list every new rule introduced. Use right-binarization (introduce helpers for the rightmost groupings first).

The rule has 4 symbols on the right-hand side. CNF requires exactly 2. We binarize by introducing auxiliary non-terminals, grouping from right to left (right-binarization):

1
Introduce X2 to cover the last two symbols: PP NP
New rule: X2 → PP NP  (already binary — two non-terminals)
2
Introduce X1 to cover NP X2
New rule: X1 → NP X2  (binary non-terminal rule)
3
Rewrite the original rule using V and X1
New rule: VP → V X1  (binary non-terminal rule)

Full CNF rule set replacing VP → V NP PP NP:

VP → V X1 (replaces the original rule) X1 → NP X2 (new auxiliary) X2 → PP NP (new auxiliary)

If V is still a non-terminal that rewrites to individual words (pre-terminal), it is already in CNF form (V → "chased"). The auxiliary symbols X1 and X2 are internal-only; they never appear in the surface string and can be stripped out during tree reconstruction (debinarization) to recover the original flat VP node.

Note on left vs. right binarization: Right-binarization creates X2 = (PP NP) first, then X1 = (NP X2). Left-binarization would create the groupings in the opposite direction. Both produce valid CNF; right-binarization is the convention adopted by most treebank-based systems.

Knowledge Check — 4 Questions

Q1. In a CFG, which symbol is conventionally used as the start symbol?

A N — the set of all non-terminal symbols
B S — by convention representing Sentence
C Σ — the terminal alphabet
D R — the set of production rules
By convention S represents Sentence and is the designated start symbol. The parser tries to derive the entire input from S by applying production rules.
S (Sentence) is the conventional start symbol. N is the set of all non-terminals, Σ is the terminal alphabet, and R is the rule set — none of these are the start symbol.

Q2. What makes a sentence "structurally ambiguous"?

A It contains words not in the grammar's vocabulary
B More than one parse tree can be derived for the same word sequence
C The grammar has no rule whose left-hand side matches the sentence
D The sentence is ungrammatical under every possible analysis
Structural ambiguity means the same string of words can be analysed with two or more different tree structures, each representing a distinct meaning — as in "I saw the man with the telescope."
Structural ambiguity arises from multiple valid parse trees — not from vocabulary gaps or ungrammaticality. The sentence is grammatical; the grammar simply permits more than one structure.

Q3. The CYK parsing algorithm requires the grammar to be in:

A Greibach Normal Form (GNF)
B Chomsky Normal Form (CNF) — rules A→BC or A→a only
C Extended Backus-Naur Form (EBNF)
D Backus-Naur Form (BNF)
CYK requires CNF — every rule must be binary (A→BC) or lexical (A→a). This enables the O(n³) dynamic programming algorithm to fill a triangular chart without ambiguity about how to split a span.
CYK requires Chomsky Normal Form (CNF), not GNF, EBNF, or BNF. CNF's strict binary-branching constraint makes the O(n³) DP table well-defined.

Q4. Which of the following is an example of subject-verb agreement violation in formal writing?

A "The dog chased the cat." — correct
B "She is happy." — correct
C "The data shows a clear trend." — "data" is grammatically plural; prescribes "show"
D "They were running fast." — correct
"Data" is the Latin plural of "datum". In formal/prescriptive grammar, the plural subject requires a plural verb: "The data show". Using the singular "shows" is a number-agreement violation under strict prescriptive rules.
Subject-verb agreement requires the verb to match the subject's number. "Data" (plural) requires "show" (plural) in formal writing. "The data shows" pairs a plural subject with a singular verb form.
