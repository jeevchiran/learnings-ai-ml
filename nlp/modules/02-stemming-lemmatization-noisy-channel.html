Morphology — Why Word Forms Matter

Even after normalisation, the same concept surfaces under many word forms: "run", "running", "runs", "ran" all refer to the same act. Morphological processing maps these variants to a single canonical form. Every word decomposes into morphemes — the smallest units of meaning.

Free morphemes

Stand alone as words: run, happy, compute.

Bound morphemes

Must attach to a root: -ing, -ed, un-, -ation.

Roots vs affixes

The root carries primary meaning; affixes attach before (prefix) or after (suffix). In uncomputerised: root = comput, affixes = un-, -er, -ise, -d.

Inflectional morphology

Marks grammatical properties (tense, number, case) without changing word class: "runs", "ran", "running" are all verbs.

Derivational morphology

Creates new words, often changing word class: compute (verb) → computation (noun). More semantically significant than inflection.

Why NLP cares

Recognising that "studying", "studied", and "studies" share the same root lets search engines, classifiers, and chatbots treat them as equivalent.

Inflectional vs. derivational — a practical rule of thumb Inflectional morphology is almost always worth collapsing: "runs" and "ran" should match "run" in most applications. Derivational morphology is more nuanced: "compute" and "computation" are related but distinct — sometimes you want them grouped (broad IR), sometimes not (precise QA). This tension shapes the stemming vs. lemmatization trade-off we explore next.

A search for "run" may need to match all of:

runningVERB runsVERB ranVERB runnerNOUN runNOUN runnableADJ

Without morphological processing, a string-match for "run" misses the other five. Stemming and lemmatization map all six to a single canonical form, improving recall.

Stemming

Stemming is a crude heuristic process that chops word endings by fixed rules, without grammar or vocabulary knowledge. The output — the stem — need not be a real word; the goal is that all inflected (and some derived) forms produce the same stem.

The Porter Stemmer

The Porter Stemmer (Porter, 1980) applies transformation rules in five sequential phases. The key concept is measure m — the number of vowel-consonant (VC) sequences in the stem — used to guard against over-stripping short words.

Measure m — quick definition

Denote a vowel as V and a consonant as C. The measure m of a word is the count of (VC) pairs in the pattern [C](VC)m[V]. For example: "tr" → m = 0; "tree" → m = 0; "troubles" → m = 1; "private" → m = 2; "oats" → m = 1. Rules guarded by "(m>0)" only fire when the remaining stem is not trivially short.

The five phases (simplified), each with representative examples:

1a
Handle plural -S endings
Rules (applied in priority order): SSES → SS ("caresses" → "caress"), IES → I ("ponies" → "poni", "ties" → "ti"), SS → SS (no change: "caress" → "caress"), S → "" ("cats" → "cat", "runs" → "run"). The IES → I rule is deliberately crude — "ponies" becomes "poni", not "pony".
1b
Handle -ED and -ING
(m>0) EED → EE ("agreed" → "agree"; "feed" → "feed" because m=0). (*v*) ED → "" if the stem contains a vowel: "plastered" → "plaster", "caressed" → "cares" (note: this is an error — the stem "cares" is misleading). (*v*) ING → "": "motoring" → "motor", "running" → "run". A cleanup sub-step then handles double consonants (e.g., "matting" → "matt" → "mat") and restores E where needed ("hoping" → "hop" + final-e rule → "hope").
2
Transform common suffixes
(m>0) apply a table of suffix substitutions: ATIONAL → ATE ("relational" → "relate"), TIONAL → TION ("conditional" → "condition"), ENCI → ENCE ("valenci" → "valence"), OUSLI → OUS ("callousli" → "callous"), ALISM → AL ("feudalism" → "feudal"), and many more.
3
Remove -icate, -ative, -alize, -iciti, -ical, -ful, -ness
(m>0): ICATE → IC ("triplicate" → "triplic"), ATIVE → "" ("formative" → "form"), ALIZE → AL ("formalize" → "formal"), ICAL → IC ("electrical" → "electr"), FUL → "" ("hopeful" → "hope"), NESS → "" ("goodness" → "good").
4–5
Remove residual derivational suffixes; clean up final -e and -ll
Phase 4 strips longer derivational endings like -ance, -ence, -ism, -able, -ible, -ion, -ment, -ous, -ive, -ize (all requiring m>1). Phase 5a removes a terminal E when m>1, or when m=1 and the preceding context is not a short vowel sequence. Phase 5b: (m>1) double-L ending → L ("controll" → "control").

Snowball (Porter2)

Snowball (also by Porter) fixes known errors (e.g., "generously" → "generous" not "generos"), handles vowel patterns more accurately, and covers many languages beyond English. NLTK ships both as PorterStemmer and SnowballStemmer.

Over-stemming and Under-stemming

Over-stemming

  • The stemmer strips too much, merging words that should remain distinct
  • Classic example: "universe", "university", "universal" → all map to "univers"
  • Also: "news" → "new" (stripping valid noun as if it were an adjective)
  • Damages precision: retrieval returns semantically unrelated documents

Under-stemming

  • The stemmer strips too little, failing to merge forms that should be grouped
  • Example: "alumnus" and "alumni" may not be conflated
  • Also: irregular forms like "mice"/"mouse" are not handled by rule-based stemmers
  • Damages recall: related documents are missed
Python · NLTK — PorterStemmer & SnowballStemmer 
from nltk.stem import PorterStemmer, SnowballStemmer

porter   = PorterStemmer()
snowball = SnowballStemmer("english")

words = ["running", "studies", "better", "caresses", "happily",
         "computing", "generously", "university", "universal"]

for w in words:
    print(f"{w:15} → Porter: {porter.stem(w):15} Snowball: {snowball.stem(w)}")

Lemmatization uses a morphological lexicon to map inflected forms to their lemma (canonical dictionary headword), always producing a valid word. It requires POS context — "better" has two lemmas:

WordNet Lemmatizer (NLTK)

NLTK's WordNetLemmatizer requires a POS argument; without it, defaults to NOUN (often wrong for verbs).

spaCy Lemmatization

spaCy integrates lemmatization into its pipeline: nlp(text) gives each token a token.lemma_, with POS predicted automatically.

The Noisy Channel Model

Beyond word-form variation, NLP must handle typos: "teh" → "the", "recieve" → "receive". The noisy channel model provides the framework: weigh how likely the intended word is (language model) against how likely this particular typo is given that word (error model).

The Metaphor

The writer has intended word w and transmits it through a noisy channel (the keyboard). The spell checker receives corrupted string e and must decode back to the most probable w.

Noisy Channel — Channel Model Derivation

We want the word w that maximises the posterior probability given the observation e. By Bayes' theorem:

\hat{w} = \underset{w \in \mathcal{V}}{\arg\max}\; P(w \mid e) = \underset{w \in \mathcal{V}}{\arg\max}\; \frac{P(e \mid w) \cdot P(w)}{P(e)}

Since P(e) is the same constant for all candidate w (the observation is fixed), we can drop it from the argmax:

\hat{w} = \underset{w \in \mathcal{V}}{\arg\max}\; P(e \mid w) \cdot P(w)

This is the noisy channel equation. Its two components are:

  • P(w) — the language model (prior): how probable is word w in this language/context? A unigram LM uses word frequency; a bigram LM conditions on the preceding word.
  • P(e \mid w) — the channel/error model (likelihood): given that the writer intended w, how probable is it that they produced the observed string e? This is estimated from a confusion matrix built from annotated typo data.

The Error Model: Confusion Matrices

P(e \mid w) is estimated from annotated typo corpora. For each edit type, we count how often a particular character pair is involved:

  • Substitution: character x typed as y (e.g., "a" → "e", keyboard adjacency)
  • Transposition: adjacent xy typed as yx (e.g., "th" → "ht", producing "teh")
  • Deletion: a character is dropped
  • Insertion: a spurious character is added

Counts are normalised to probabilities; the overall P(e \mid w) is the product of per-edit probabilities along the optimal edit path.

Candidate Generation

Candidates are all dictionary words within edit distance 1 or 2 of the observed string (Module 1). About 80% of non-word errors fall within edit distance 1 (Damerau, 1964).

Real-word errors This model only handles non-word errors (typos not in the dictionary, e.g., "teh"). It cannot correct real-word errors ("their" → "there") — those require a language model that detects contextual implausibility, covered in later modules.

Problem 2: Noisy Channel Calculation

Medium

You observe the misspelled word "recieve". Three candidate corrections are proposed with the following scores:

Candidate wP(w) — language modelP(e|w) — error model
receive0.010.85
relieve0.0030.05
retrieve0.0020.02

Which word does the noisy channel model select? Show the full calculation for each candidate, and write the argmax formula.

The noisy channel model selects:

\hat{w} = \underset{w}{\arg\max}\; P(e \mid w) \cdot P(w)

Computing the joint score for each candidate:

\text{receive:} \quad P(e \mid w) \cdot P(w) = 0.85 \times 0.01 = \mathbf{0.0085}
\text{relieve:} \quad P(e \mid w) \cdot P(w) = 0.05 \times 0.003 = 0.00015
\text{retrieve:} \quad P(e \mid w) \cdot P(w) = 0.02 \times 0.002 = 0.00004

Ranking: receive (0.0085) > relieve (0.00015) > retrieve (0.00004).

The model selects "receive" with a score of 0.0085 — nearly 57× higher than the next candidate.

Interpretation: "receive" wins on both components — it is common and the "ie/ei" transposition is a very frequent confusion. "relieve" and "retrieve" score poorly on the error model as they require more/different edits.

Problem 3: Stemming Failure Analysis

Hard

The Porter stemmer produces "univers" as the stem for all of: "university", "universe", and "universal".

(a) Is this an instance of over-stemming or under-stemming? Justify your answer precisely.

(b) Give a concrete example of an IR query where this conflation is beneficial, and a concrete example where it is harmful.

(c) Propose a principled test to help decide whether to use stemming or lemmatization for a given application. Your test should be actionable — something a practitioner could actually run.

(a) This is over-stemming. Over-stemming occurs when distinct words that should not be conflated are mapped to the same stem. "Universe" (a cosmological entity), "university" (an educational institution), and "universal" (an adjective meaning applicable to all) are semantically distinct. Their shared Latin root universus is an etymological coincidence that does not make them synonyms in modern English. By merging them, the stemmer is asserting a semantic equivalence that does not exist. Compare under-stemming, which would fail to merge "running" and "run" — an error in the opposite direction.

(b) Beneficial example: A broad academic IR query like "universal access to higher education". If the user is researching education policy, they may also want results containing "university access" or "universities universal". Conflating these three terms under "univers" increases recall — retrieving more potentially relevant documents — at the cost of some precision.

Harmful example: A scientific query distinguishing cosmology from education policy, such as "universe expansion models". After stemming, documents about "university curriculum expansion" and "universal healthcare models" share the same "univers" token and will be scored as relevant. This pollutes the result set with semantically unrelated material — a precision failure.

(c) Principled test — precision/recall audit on a held-out query set:

  1. Collect 50–100 representative queries and their relevance judgements (which documents are truly relevant to each query).
  2. Index the same document corpus twice: once with stemming, once with lemmatization.
  3. Compute precision@10 and recall@10 for each system on every query.
  4. Compare aggregate P@10 and R@10 across both systems. Also look at per-query deltas — some queries will be harmed by stemming (over-conflation), others will be helped (better recall). If the task has strict precision requirements (e.g., a legal or medical search system), choose the system with higher P@10. If recall is paramount (e.g., patent prior-art search), prefer higher R@10.
  5. As a quick proxy: count "stem-collapse collisions" — pairs of semantically distinct words that map to the same stem — in the target domain vocabulary. A high collision rate predicts precision degradation.

A Indexing billions of web pages for a large-scale search engine where speed is critical

Correct. QA and NLU tasks benefit from lemmatization's precision.
