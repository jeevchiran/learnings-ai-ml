Named Entity Recognition (NER)

Named Entity Recognition (NER) identifies expressions that refer directly to real-world objects — people, companies, countries, dates — and classifies them into predefined categories. Unlike POS tagging, NER focuses on spans of tokens that form a single meaningful reference: "New York" is one GPE, "Apple" the company is one ORG.

Entity types in standard datasets

The most widely used NER benchmarks — CoNLL 2003 and OntoNotes 5.0 — define rich taxonomies of entity types:

Core entity types (CoNLL 2003)
  • PERSON — people, fictional or real
  • ORG — companies, agencies, institutions
  • GPE — geopolitical entities (countries, cities, states)
  • LOC — non-GPE locations (mountains, rivers)
  • MISC — nationalities, products, events
Extended types (OntoNotes)
  • DATE, TIME — temporal expressions
  • MONEY, PERCENT — numeric values
  • PRODUCT — objects, vehicles, software
  • EVENT — named events (wars, elections)
  • LAW — named legal documents

Why NER matters

NER is a foundational pre-processing step for a wide range of downstream tasks:

  • Information extraction — identifying who did what to whom in news articles
  • Question answering — entity types constrain answer candidates ("Who founded Tesla?" expects a PERSON)
  • Summarization — entities anchor the key facts in a document
  • Relation extraction — you can only extract [ORG] acquired [ORG] for [MONEY] if you first know what is an ORG and what is MONEY
  • Knowledge base population — linking mentions to structured databases like Wikidata

NER as sequence labeling

NER is a span identification problem: a single entity may span multiple tokens ("Barack Obama" = one PERSON, "New York City" = one GPE).

Example: entity spans in a sentence

"Barack Obama visited Apple's headquarters in Cupertino on Monday."

→ Barack Obama visited Apple's headquarters in Cupertino on Monday.

To handle multi-token entities in a per-token framework, we need a span encoding scheme — IOB tagging, covered next.

spaCy NER in practice

Python — NER with spaCy 
import spacy

nlp = spacy.load("en_core_web_sm")
text = "Barack Obama visited Apple's headquarters in Cupertino on Monday."
doc = nlp(text)

for ent in doc.ents:
    print(f"{ent.text:25} → {ent.label_:8} ({spacy.explain(ent.label_)})")
Barack Obama → PERSON (People, including fictional) Apple → ORG (Companies, agencies, institutions, etc.) Cupertino → GPE (Countries, cities, states) Monday → DATE (Absolute or relative dates or periods)

IOB / BIO Tagging Scheme

The BIO scheme (IOB2) converts span-level annotation into per-token labels:

B
B-TYPE — Beginning of entity
The first token of an entity of type TYPE. Every entity span starts with exactly one B tag. Example: "Barack" receives B-PERSON.
I
I-TYPE — Inside (continuation) of entity
Any token after the first within the same entity span. "Obama" in "Barack Obama" receives I-PERSON. An I tag may only follow a B or another I of the same TYPE.
O
O — Outside any entity
Tokens that are not part of any named entity. Verbs, prepositions, punctuation, and most common nouns receive O. An I tag may never follow O.

BIO in action

The sentence "Barack Obama visited New York" would be labeled:

BarackB-PER ObamaI-PER visitedO NewB-GPE YorkI-GPE

BIOES extension

For richer encoding, the BIOES scheme adds two more tag types:

  • E-TYPE — End of entity (the final token of a multi-token span)
  • S-TYPE — Single token entity (an entity that is exactly one token)

BIOES lets a model directly distinguish one-token entities (S) from the start of a multi-token entity (B). BIO remains dominant in practice due to its simplicity.

Why IOB matters for modeling Converting span labeling to BIO tagging turns NER into per-token classification. This makes it amenable to any sequence classifier — HMMs, CRFs, BiLSTM-CRFs, or BERT fine-tuning — while preserving span boundary information. The alternative (predicting start and end indices) is used in span-extraction QA but is harder to optimize as a sequence model.

spaCy exposes IOB labels directly on each token:

Python — NER as IOB labeling 
for token in doc:
    iob = token.ent_iob_
    etype = token.ent_type_ if token.ent_type_ else "O"
    tag = f"{iob}-{etype}" if iob != "O" else "O"
    print(f"{token.text:15} {tag}")
Barack B-PERSON Obama I-PERSON visited O Apple's B-ORG headquarters O in O Cupertino B-GPE on O Monday B-DATE . O

Conditional Random Fields (CRF) for Sequence Labeling

A simple per-token classifier predicts each label in isolation, but BIO has hard structural constraints a per-token model cannot enforce:

  • I-PERSON can never follow O
  • I-PERSON can never follow B-ORG or I-ORG
  • An entity span, once started with B-X, can only be continued with I-X

What is a CRF?

A Conditional Random Field (CRF) models the conditional distribution of an entire label sequence given the input sequence:

P(\mathbf{y} \mid \mathbf{x}) = \frac{\exp\!\Bigl(\displaystyle\sum_{i=1}^{n} \sum_{k} \lambda_k f_k(y_{i-1},\, y_i,\, \mathbf{x},\, i)\Bigr)}{Z(\mathbf{x})}

where Z(\mathbf{x}) normalizes over all label sequences, f_k are feature functions over (y_{i-1}, y_i, \mathbf{x}, i), and \lambda_k are learned weights.

Feature functions

Typical NER features include:

Observation features
  • Word identity: is the word "Obama"?
  • Prefix/suffix: ends in "-son", "-tion"
  • Capitalization: first letter uppercase?
  • Digit pattern: contains digits?
  • Previous/next word identity
  • POS tag of current token
Transition features
  • Label pair (y_{i-1}, y_i)
  • e.g., f = 1 if y_{i-1}=B-PER and y_i=I-PER
  • Learned weight → high for valid transitions
  • Weight → -∞ for impossible transitions
  • (O → I-PER is impossible: weight is very negative)

Decoding: Viterbi algorithm

The Viterbi algorithm finds the highest-scoring label sequence via dynamic programming. At each position i, it computes the best score for reaching label y_i:

V_i(y) = \max_{y'} \Bigl[ V_{i-1}(y') + \sum_k \lambda_k f_k(y', y, \mathbf{x}, i) \Bigr]

This matches the HMM Viterbi recurrence — both are linear-chain graphical models.

CRF vs. HMM

CRF (discriminative)

  • Models P(\mathbf{y} \mid \mathbf{x}) directly
  • Can use arbitrary, overlapping features
  • Features can look at the whole input sequence
  • Generally higher accuracy on NER benchmarks
  • Training: maximize conditional log-likelihood

HMM (generative)

  • Models P(\mathbf{x}, \mathbf{y}) jointly
  • Features: emission and transition probabilities only
  • Observations assumed independent given labels
  • Simpler to train (count and normalize)
  • Can sample sequences; useful for generation tasks
Modern NER: BiLSTM-CRF The standard architecture for high-accuracy NER combines a bidirectional LSTM (which reads the sentence left-to-right and right-to-left to extract rich contextual features for each token) with a CRF output layer (which enforces valid label transitions globally). The BiLSTM replaces hand-crafted feature functions; the CRF replaces a simple softmax output. Transformer-based models (BERT, RoBERTa) have largely superseded this, but the CRF output layer is still commonly added even on top of transformers for structured prediction.

Semantic Role Labeling (SRL)

Semantic Role Labeling (SRL) identifies the predicate–argument structure of each verb — who did what to whom, when, where, and how. Two annotation frameworks underlie most SRL work:

  • FrameNet (Fillmore) — frames are event types with specific semantic roles (frame elements). The frame "Commerce_buy" has roles Buyer, Goods, Seller, Money.
  • PropBank — more verb-centric; uses numbered arguments (ARG0, ARG1, …) plus adjuncts.

PropBank core roles

Semantic roles (PropBank)

RoleSemantic meaningExample
ARG0Agent / doer of the action"The dog chased the cat."
ARG1Patient / theme (entity acted upon)"The dog chased the cat."
ARG2Instrument / beneficiary / attribute"She cut the bread with a knife."
ARGM-TMPTemporal modifier"He left yesterday."
ARGM-LOCLocation modifier"They met in the garden."
ARGM-MNRManner modifier"She ran quickly."
ARGM-CAUCause modifier"He left because of the noise."

SRL example

Predicate–argument structure

"The dog chased the cat in the garden yesterday."

Predicate: chased

  • ARG0 The dog — who did the chasing
  • ARG1 the cat — who was chased
  • ARGM-LOC in the garden — where
  • ARGM-TMP yesterday — when

A sentence can have multiple predicates: "She bought a coffee and left quickly" yields bought (ARG0=She, ARG1=a coffee) and left (ARG0=She, ARGM-MNR=quickly).

SRL vs. NER

Key distinction: SRL vs. NER NER classifies expressions by their type (PERSON, ORG, GPE) — the label is absolute, independent of context. SRL assigns roles relative to a predicate — the same expression "the dog" is ARG0 of "chased" but might be ARG1 of "was bathed". SRL is about event semantics; NER is about entity typing.
Python — Approximating SRL via dependency parse (spaCy) 
import spacy

        print(f"Predicate: {token.text}")
        for child in token.children:
            print(f"  {child.dep_:10} → {child.text}")

Coreference Resolution

Coreference resolution clusters all mentions in a document that refer to the same real-world entity, even when expressed differently.

Example: three mentions, one person

"Barack Obama gave a speech on Tuesday. He said the policy would benefit working families. The President's team agreed with the assessment."

Chain: {Barack Obama, He, The President} — all refer to the same real-world individual.

Types of referring expressions

  • Proper nouns — "Barack Obama", "Apple", "Paris"
  • Pronouns — "he", "she", "it", "they", "his", "her"
  • Definite noun phrases — "the president", "the company", "the city"
  • Demonstratives — "this approach", "that result", "these findings"
  • Zero anaphora — common in pro-drop languages (Japanese, Spanish): the subject is omitted but understood

Coreference vs. anaphora

Coreference vs. anaphora Anaphora: an expression depends on another for interpretation ("He apologized" — "He" depends on a prior antecedent). Coreference: two expressions refer to the same entity ("the author of Hamlet" and "Shakespeare"). Anaphora is a discourse relation; coreference is ontological. Most pronouns are both, but not all coreferences involve anaphora.

Coreference algorithms

1
Hobbs algorithm (1978) — rule-based pronoun resolution
Tree-traversal algorithm searching left-to-right and upward for the most plausible antecedent. Fast and interpretable, but limited to pronouns and requires accurate parse trees.
2
Stanford CoreNLP sieve (2010) — deterministic multi-pass
Cascade of increasingly permissive sieves (exact string match → head word match → pronoun resolution). High precision at the cost of recall; still a strong rule-based baseline.
3
Neural span-based models — end-to-end
Lee et al. (2017): jointly learning mention detection and coreference scoring end-to-end outperforms pipelines. Encodes candidate spans with BiLSTM or BERT, then scores all span pairs. State of the art on OntoNotes.

Problem 1: BIO Tagging by Hand

Easy

Apply BIO tagging to the following sentence. Entity spans are given:

"Apple CEO Tim Cook announced a new iPhone in San Francisco."

Entity spans: [Apple = ORG], [Tim Cook = PERSON], [iPhone = PRODUCT], [San Francisco = GPE]

Write the full BIO sequence — one token/tag per token.

AppleB-ORG CEOO TimB-PER CookI-PER announcedO aO newO iPhoneB-PROD inO SanB-GPE FranciscoI-GPE .O

Note: single-token entities (Apple, iPhone) use only B (no I). Multi-token spans (Tim Cook, San Francisco) use B then I. In BIOES, single-token entities would be S-ORG / S-PRODUCT and multi-token spans would end with E-PERSON / E-GPE.

Problem 2: CRF Transition Constraints

Medium

In a linear-chain CRF for NER with the BIO tagset {O, B-PER, I-PER, B-ORG, I-ORG}, list all invalid tag transitions — transitions that should be assigned effectively zero (or -\infty) weight. Explain briefly why each is invalid.

Invalid transitions (8 total):

TransitionWhy invalid
O → I-PERAn I tag must be preceded by B of the same type. Cannot start an I-PER from outside an entity.
O → I-ORGSame reason — cannot start I-ORG from O.
B-PER → I-ORGI-ORG would continue an ORG entity, but we just started a PER entity with B-PER. Types must match.
B-ORG → I-PERI-PER would continue a PER entity, but the preceding tag opens an ORG entity. Type mismatch.
I-PER → I-ORGCannot switch entity type mid-span. I-PER must be followed by I-PER (to continue) or close the entity.
I-ORG → I-PERSame — cannot switch from ORG span to PER span without a B tag.
B-PER → I-ORG(Already listed — type mismatch)
I-ORG → I-PER(Already listed — type mismatch)

The pattern: an I-X tag may only follow B-X or I-X (same type X). Any transition to I-X from anything other than B-X or I-X is invalid. For k entity types, there are 2k^2 - 2k invalid transitions (each of the k I-types cannot follow O or any of the k-1 B/I-tags of different types, minus the valid self-continuations).

In practice, the CRF learns very large negative weights for these transitions; the Viterbi decoder then never selects them.

Problem 3: Coreference Chain Analysis

Hard

Consider the following passage:

"The committee approved the proposal. Its members were satisfied. They believed it would benefit the company."

Identify all coreference chains. For each chain: (a) list the referring expressions, (b) classify each by type (proper noun, pronoun, definite NP, possessive NP), (c) flag any cases of genuine ambiguity and explain why.

Chain 1 — The committee:

  • "The committee" — definite NP (antecedent)
  • "Its" (in "Its members") — possessive pronoun; refers to the committee
  • "They" — plural pronoun; refers to the committee/its members (see ambiguity note below)

Chain 2 — The proposal:

  • "the proposal" — definite NP (antecedent)
  • "it" (in "it would benefit") — third-person singular pronoun; most naturally refers to the proposal

Ambiguity analysis:

  • "They" — could refer to (a) "The committee" (treating a committee as a plural group) or (b) "Its members" (the more proximate plural NP). Both are semantically plausible. In English, collective nouns like "committee" allow plural pronoun agreement. A coreference resolver must decide whether "They" links to the same chain as "The committee" or starts a sub-chain via "Its members".
  • "it" — could refer to "the proposal" or abstractly to "the committee's decision to approve." In context, "the proposal" is the more salient referent and is the standard interpretation.
  • "Its" is unambiguous: the only available singular possessive antecedent is "The committee".

Q1. In BIO tagging, what does the "I" tag signify?

A The token is the Initial (first) token of any sentence.
B The token is Inside (a continuation of) an entity span already started with B.
C The token is an Important named entity standing alone.
D The token is of unknown or ambiguous type.
B = Beginning, I = Inside (continuation), O = Outside. "I" only ever follows a B of the same entity type — enforcing that entity spans are contiguous.
BIO stands for Beginning, Inside, Outside. "I" marks continuation tokens of an entity already opened by a "B" tag of the same type. It cannot follow "O".

Q2. CRFs are preferred over independent per-token classifiers for NER because:

A CRFs are faster to train and require less data.
B CRFs model dependencies between adjacent labels, enforcing valid BIO tag sequences globally.
C CRFs require no training data — they are purely rule-based.
D CRFs automatically discover entity types without supervision.
Correct. A per-token softmax might independently predict I-PER after O (illegal in BIO). A CRF scores entire label sequences, allowing it to learn — and enforce — that I-PER must follow B-PER or I-PER, never O.
The key advantage is label dependency modeling. CRFs score the joint probability of a label sequence P(y|x), not individual labels in isolation. This lets the CRF learn that certain label transitions are impossible in BIO.

Q3. Semantic Role Labeling (SRL) identifies which of the following?

A The entity types of proper nouns in a sentence (PERSON, ORG, etc.).
B The predicate–argument structure: who did what to whom, when, and where.
C The co-referential expressions that refer to the same real-world entity.
D The syntactic chunks (NP, VP, PP) in a sentence.
Correct. SRL assigns semantic roles (ARG0=agent, ARG1=patient, ARGM-LOC=location, etc.) to constituents relative to each predicate verb — revealing the event structure of the sentence.
NER identifies entity types (A); coreference resolution links co-referential mentions (C); chunking finds phrase structure (D). SRL is about event semantics: what role each phrase plays relative to the predicate.

Q4. In "The scientists published their results. They were surprised by the findings." — "they" refers to:

A The results.
B The scientists.
C It is genuinely ambiguous — either referent is equally plausible.
D "Their" from the first sentence (pronouns resolve to possessives).
Correct. "They" is a plural pronoun. "The scientists" is the plural human agent — by world knowledge, scientists can be surprised by findings. "Results" is also plural but semantically implausible as the subject of "surprised by the findings" (results cannot be surprised).
"They" must resolve to a plural noun phrase. Both "the scientists" and "results" are plural, but only "the scientists" is semantically plausible as the subject of "were surprised." Semantic plausibility, not just number agreement, drives coreference.
