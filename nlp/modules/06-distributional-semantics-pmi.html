The Distributional Hypothesis

Bridge from Module 5. WordNet is hand-crafted and hard to scale. Can we instead learn meaning automatically from raw text?

J.R. Firth answered this in 1957:

"You shall know a word by the company it keeps."
— J.R. Firth, A Synopsis of Linguistic Theory (1957)

The distributional hypothesis: words appearing in similar contexts have similar meanings — shifting the question from "what does a word mean?" to "what company does it keep?"

Intuition: 'dog' and 'cat'

dog and cat both appear near pet, feed, vet, furry — and almost never near photosynthesis or transistor. Similar context distributions correctly predict semantic relatedness. dog and democracy share almost no context words, correctly predicting semantic distance.

This is the foundation of modern word representations: Word2Vec, GloVe, FastText, and LLM input layers all exploit the distributional hypothesis. PMI and co-occurrence matrices are the prerequisite for understanding why they work.

No labeling required

Fully unsupervised — any raw text corpus suffices, no annotation costs.

Language-agnostic

The same algorithm works for English, Swahili, or ancient Greek without language-specific rules.

Domain-adaptable

Train on medical text and "discharge" carries a clinical meaning; train on electrical engineering text and it shifts accordingly.

Scalable

More data almost always helps — richer co-occurrence statistics yield higher-quality representations.

Co-occurrence Matrices

A co-occurrence matrix has target words as rows, context words as columns, and co-occurrence counts as cells.

Window-Based Co-occurrence

Given a sentence and a window size k, every pair of words within distance k (positions i-k to i+k, excluding position i itself) is counted as a co-occurrence. Common choices are k = 2 (tight syntactic context) or k = 5 (broader topical context).

Building a Co-occurrence Matrix: Step by Step

Consider this small four-sentence corpus with window size k = 2:

"I like dogs and cats"
"Dogs are loyal pets"
"Cats are independent pets"
"I prefer dogs over cats"

Target vocabulary (alphabetical): and, are, cats, dogs, i, independent, like, loyal, over, pets, prefer

For sentence 1, "I like dogs and cats", with k=2:

  • dogs co-occurs with: i (dist 2), like (dist 1), and (dist 1), cats (dist 2)
  • cats co-occurs with: like (dist 2 — wait, dist 3 — outside window), dogs (dist 2), and (dist 1)

Repeat for all sentences, accumulate counts. A subset of the resulting matrix (showing key words):

Target \ Context dogs cats pets loyal independent i are
dogs 0 3 2 1 0 2 1
cats 3 0 2 0 1 1 1
pets 2 2 0 1 1 0 2
loyal 1 0 1 0 0 0 1

dogs and cats have similar row vectors — the distributional hypothesis made numerical.

Problem: Raw counts are dominated by function words. "the", "is", "and" appear everywhere, so "dog–the" has enormous counts despite carrying no semantic information. This motivates PMI weighting.

Python: Building the Co-occurrence Matrix

import numpy as np
from collections import defaultdict

def build_cooccurrence(sentences, window=2):
    # Build vocabulary
    all_words = [w for sent in sentences for w in sent]
    vocab = sorted(set(all_words))
    word2idx = {w: i for i, w in enumerate(vocab)}

    # Count co-occurrences
    cooc = np.zeros((len(vocab), len(vocab)))
    for sent in sentences:
        for i, w in enumerate(sent):
            for j in range(max(0, i - window), min(len(sent), i + window + 1)):
                if i != j:
                    cooc[word2idx[w], word2idx[sent[j]]] += 1
    return cooc, vocab, word2idx

sentences = [
    ["i", "like", "dogs", "and", "cats"],
    ["dogs", "are", "loyal", "pets"],
    ["cats", "are", "independent", "pets"],
    ["i", "prefer", "dogs", "over", "cats"]
]
cooc, vocab, w2i = build_cooccurrence(sentences)
print(vocab)
# ['and', 'are', 'cats', 'dogs', 'i', 'independent', 'like', 'loyal', 'over', 'pets', 'prefer']

Pointwise Mutual Information (PMI)

Pointwise Mutual Information measures how much more often two words co-occur than expected under independence.

The PMI Formula

1
Define joint and marginal probabilities
From the co-occurrence matrix, estimate:
P(x, y) = \frac{\text{count}(x, y)}{N} \qquad P(x) = \frac{\text{count}(x)}{N} \qquad P(y) = \frac{\text{count}(y)}{N}
where N is the total number of co-occurrence tokens (sum of all cells).
2
Compute PMI
\text{PMI}(x, y) = \log_2 \frac{P(x, y)}{P(x) \cdot P(y)}
If x and y are independent, P(x,y) = P(x)P(y) and PMI = 0. Positive PMI means they attract; negative means they repel.
3
Clip to PPMI
\text{PPMI}(x, y) = \max\bigl(0,\; \text{PMI}(x, y)\bigr)
Negative PMI values are unreliable for rare pairs (where P(x,y) \approx 0 gives PMI \to -\infty). Clipping at 0 — Positive PMI — discards unreliable negatives.

Interpreting the PMI Score

PMI > 0 — Attraction

Co-occurs more than chance. Strong semantic association (e.g., PMI("ice", "cream") is large and positive).

PMI = 0 — Independence

Co-occurs exactly as often as chance predicts. No detectable semantic relationship.

PMI < 0 — Repulsion

Co-occurs less than chance. Zero counts drive PMI to -\infty; PPMI clips these to 0.

Key insight: PMI vs. raw counts. "ice" and "the" co-occur enormously in any large corpus, but PMI("ice","the") ≈ 0 because "the" appears near every word by chance. PMI("ice","cream") is large precisely because that pairing exceeds chance expectation.

Python: Computing PPMI

def compute_ppmi(cooc):
    total = cooc.sum()
    row_sums = cooc.sum(axis=1, keepdims=True)
    col_sums = cooc.sum(axis=0, keepdims=True)

    # P(x,y), P(x), P(y)
    pxy = cooc / total
    px  = row_sums / total
    py  = col_sums / total

    # PMI = log2(P(x,y) / P(x)*P(y)); clip negatives to 0 for PPMI
    with np.errstate(divide='ignore', invalid='ignore'):
        pmi = np.log2(pxy / (px * py))
    pmi  = np.nan_to_num(pmi, nan=0.0, neginf=0.0)
    ppmi = np.maximum(0, pmi)   # PPMI
    return ppmi

ppmi = compute_ppmi(cooc)
# ppmi[w2i['dogs'], w2i['loyal']] will be positive
# ppmi[w2i['dogs'], w2i['independent']] will be 0.0 (no co-occurrence)

Building and Interpreting the PMI Matrix

Each word is now a high-dimensional sparse vector. Semantic similarity is measured with cosine similarity.

Cosine Similarity

For two word vectors \mathbf{u} and \mathbf{v}:

\text{cosine\_sim}(\mathbf{u}, \mathbf{v}) = \frac{\mathbf{u} \cdot \mathbf{v}}{\|\mathbf{u}\| \cdot \|\mathbf{v}\|} = \frac{\displaystyle\sum_i u_i v_i}{\sqrt{\displaystyle\sum_i u_i^2} \cdot \sqrt{\displaystyle\sum_i v_i^2}}

Range: [-1, +1]; for PPMI vectors (all non-negative), [0, 1]. Preferred over Euclidean distance because it is invariant to vector magnitude — a word appearing twice as often gets a longer vector but the same semantic direction.

Finding the Most Similar Word

Given PPMI matrix M:

\text{similar\_to}(\text{"dogs"}) = \underset{w \neq \text{"dogs"}}{\arg\max}\; \text{cosine\_sim}(M_{\text{dogs}},\, M_w)

On the four-sentence corpus above, this returns "cats" — they share the same high-PMI context words (pets, loyal/independent, "i", "prefer").

Limitations of Raw PPMI Vectors

  • High-dimensional: With a 50,000-word vocabulary, vectors have 50,000 dimensions — expensive to store and compare.
  • Sparse: Most entries are 0 (most word pairs never co-occur). Sparse vectors are inefficient for neural networks.
  • No generalisation: Two words can have identical meanings but never share a context word in the corpus — they'll appear unrelated.
SVD compresses sparse PPMI vectors into dense low-dimensional ones; word2vec (Module 9) learns dense vectors directly, implicitly recovering a structure similar to PPMI at much larger scale.

Practice

Problem 1: PMI Calculation

Easy

Given co-occurrence statistics from a corpus:

  • count(dogs, pets) = 3
  • count(dogs) = 8     count(pets) = 5
  • Total co-occurrence tokens N = 50

Compute PMI(dogs, pets). Is the value positive or negative? What does this tell you about the semantic relationship?

Compute the three probabilities:

P(\text{dogs}, \text{pets}) = \frac{3}{50} = 0.06
P(\text{dogs}) = \frac{8}{50} = 0.16 \qquad P(\text{pets}) = \frac{5}{50} = 0.10

Now apply the PMI formula:

\text{PMI}(\text{dogs}, \text{pets}) = \log_2 \frac{0.06}{0.16 \times 0.10} = \log_2 \frac{0.06}{0.016} = \log_2 3.75 \approx 1.91

Positive — "dogs" and "pets" co-occur 3.75× more than chance, indicating strong semantic association.

Problem 2: PPMI Vectors and Cosine Similarity

Medium

Two words have PPMI vectors over a 5-word vocabulary:

\mathbf{w}_1 = [0.5,\; 1.2,\; 0.0,\; 0.8,\; 0.3]
\mathbf{w}_2 = [0.4,\; 1.1,\; 0.2,\; 0.9,\; 0.0]

Compute their cosine similarity. Are these words semantically similar?

Step 1: Dot product

\mathbf{w}_1 \cdot \mathbf{w}_2 = (0.5)(0.4) + (1.2)(1.1) + (0)(0.2) + (0.8)(0.9) + (0.3)(0) = 0.20 + 1.32 + 0 + 0.72 + 0 = 2.24

Step 2: Norms

\|\mathbf{w}_1\| = \sqrt{0.25 + 1.44 + 0 + 0.64 + 0.09} = \sqrt{2.42} \approx 1.556
\|\mathbf{w}_2\| = \sqrt{0.16 + 1.21 + 0.04 + 0.81 + 0} = \sqrt{2.22} \approx 1.490

Step 3: Cosine similarity

\text{cosine\_sim}(\mathbf{w}_1, \mathbf{w}_2) = \frac{2.24}{1.556 \times 1.490} = \frac{2.24}{2.318} \approx 0.966

Cosine of 0.966 (max 1.0) — these words are highly semantically similar, appearing in nearly identical contexts.

Problem 3: Alternative Weighting Schemes Beyond PPMI

Hard

Raw co-occurrence counts are dominated by frequent function words. PPMI is one remedy — but it is not the only one. List three alternative weighting schemes beyond PPMI, explain the rationale for each, and write the mathematical formula for one of them.

1. TF-IDF applied to co-occurrence

Treat each target word as a "document" and context words as "terms". IDF down-weights context words appearing near many targets (like stop words) and up-weights specific ones.

2. Shifted PPMI (SPPMI)

Subtract a shift \log_2 k, retaining only high-confidence positive associations:

\text{SPPMI}_k(x, y) = \max\!\left(0,\; \log_2 \frac{P(x, y)}{P(x)\,P(y)} - \log_2 k\right)

Directly analogous to word2vec's negative sampling objective (k = number of negative samples).

3. Sublinear count scaling

Replace \text{count}(x, y) with \log(1 + \text{count}(x, y)) or \sqrt{\text{count}(x, y)} before computing PMI. The 50th co-occurrence adds less information than the 1st; sublinear scaling compresses dynamic range without discarding frequent pairs.

Quiz

Q1. The distributional hypothesis states that:

Words can be distributed across documents uniformly.
Words occurring in similar contexts tend to have similar meanings.
Rare words have more specific meanings than common ones.
Every word has a unique distributional fingerprint unrelated to others.
Correct. Firth's "you shall know a word by the company it keeps" — context predicts meaning, enabling unsupervised semantic learning. The distributional hypothesis links meaning to context: similar contexts → similar meanings. This is the foundation of all distributional word representations.

Q2. PMI(x, y) < 0 means:

The two words are antonyms.
x and y co-occur less often than expected if they were independent.
The words never co-occur.
There is a data error in the co-occurrence matrix.
Correct. Negative PMI means the observed co-occurrence P(x,y) is below the expected P(x)×P(y) — the words tend to avoid each other. PMI<0 means co-occurrence is below random chance expectation — not that they never co-occur, just less often than chance predicts.

Q3. Why is PPMI preferred over raw PMI for sparse NLP data?

PPMI is faster to compute.
Negative PMI values are unreliable for rare word pairs with zero or very low counts.
PPMI eliminates all stop words.
PPMI guarantees cosine similarity equals 1 for similar words.
Correct. When two words never co-occur, PMI = −∞ (log of 0). Clipping at 0 (PPMI) prevents these unreliable negative values from dominating similarity calculations. The issue is reliability: PMI=−∞ for zero-count pairs is not meaningful. PPMI clips these to 0, making vectors more stable for downstream similarity.

Q4. Cosine similarity is preferred over Euclidean distance for comparing word vectors because:

It is faster to compute.
It is invariant to the magnitude (length) of the vectors, focusing only on direction.
It always returns a value between 0 and 1.
It works only for binary vectors.
Correct. Cosine ignores vector magnitude — a word appearing 10× more in a corpus has a longer vector but the same semantic direction. Euclidean distance would wrongly penalise this. Cosine similarity measures the angle between vectors, not their length. This makes it robust to frequency differences in the corpus. Note: it can be negative for non-PPMI vectors.
